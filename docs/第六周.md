## 4. 为什么 Embedding 是 RAG 的前置工程

### 4.1 Chat Model 和 Embedding Model 的区别

| 类型              | 输入                | 输出                   | 用途   |
| --------------- | ----------------- | -------------------- | ---- |
| Chat Model      | 用户问题、上下文 messages | 一段自然语言回答             | 生成回答 |
| Embedding Model | 一段文本              | 一个数字向量 `list[float]` | 语义检索 |

例如：

```text
文本：
FastAPI 可以通过 exception_handler 注册全局异常处理器。

Embedding 后：
[0.0123, -0.0456, 0.0789, ...]
```

Chat Model 像“会说话的大脑”。

Embedding Model 像“把文本转成坐标的编码器”。

---

### 4.2 为什么 Ollama Chat Model 不等于 Embedding Model

你现在的 `OllamaProvider` 是面向聊天的：

```text
messages -> answer
```

Embedding 要的是：

```text
text -> vector
```

即使 Ollama 也能提供 embedding 接口，概念上仍然是两个不同 Provider：

```text
LLMProvider:
  chat()
  stream_chat()

EmbeddingProvider:
  embed_text()
  embed_texts()
  dimension()
```

不要把它们混在一起。否则后面 RAG、Agent、评估都会难拆。

---

### 4.3 为什么 chunk 要转向量

传统关键词搜索只能匹配字面词。

例如用户问：

```text
FastAPI 如何统一处理错误？
```

文档里写的是：

```text
FastAPI 可以通过 exception_handler 注册全局异常处理器。
```

关键词不完全一样，但语义接近。

Embedding 的作用是把它们变成向量后，让语义接近的文本在向量空间里距离更近。

---

### 4.4 什么是向量维度

如果模型输出：

```python
[0.1, -0.2, 0.03]
```

这是 3 维向量。

如果模型输出 384 个数字，就是 384 维向量。

本周推荐模型：

```text
sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
```

通常输出 384 维向量。

所以数据库要写：

```sql
embedding vector(384)
```

`vector(384)` 的意思是：

```text
这个字段只能保存 384 维向量。
```

如果模型输出 768 维，但数据库是 `vector(384)`，写入会失败。

---

### 4.5 cosine similarity 是什么

cosine similarity 看的是两个向量方向是否接近。

```text
方向越接近，语义越相似。
方向差距越大，语义越不相似。
```

pgvector 中：

```sql
embedding <=> query_vector
```

表示 cosine distance。官方文档说明 pgvector 支持 `<->` L2 distance、`<#>` inner product、`<=>` cosine distance，并且 cosine similarity 可以用 `1 - cosine distance` 计算。([GitHub][1])

所以：

```text
distance 越小越相似
score = 1 - distance
score 越大越相似
```

## 5. 核心概念

### 5.1 Embedding

Embedding 是把文本转换成数字向量。

```python
"text" -> [0.012, -0.034, ...]
```

真实 AI 工程里，Embedding 主要用于：

```text
语义搜索
推荐
聚类
去重
RAG 检索
```

---

### 5.2 Vector

Vector 就是数字列表。

Python 中表示为：

```python
list[float]
```

多个文本的向量表示为：

```python
list[list[float]]
```

类比 JS：

```ts
number[]
number[][]
```

---

### 5.3 pgvector

pgvector 是 PostgreSQL 的向量扩展。官方说明它可以在 Postgres 中存储向量，并支持精确/近似最近邻搜索、L2、inner product、cosine distance 等距离计算。([GitHub][1])

本项目选择 pgvector 的原因：

```text
1. PostgreSQL 本身是成熟业务数据库。
2. metadata、document_id、chunk_index 可以和 embedding 放在一起。
3. 中小规模 RAG 足够用。
4. 后续可以继续扩展成正式数据库存储。
```

---

### 5.4 VectorStore

VectorStore 是“向量存储层”。

本项目中：

```text
PgVectorStore
  负责 PostgreSQL + pgvector 的读写
```

它不负责：

```text
加载模型
调用 SentenceTransformer
处理 API 请求
调用 LLM
```

---

### 5.5 EmbeddingProvider

EmbeddingProvider 是向量模型抽象。

```python
class EmbeddingProvider(ABC):
    def embed_text(self, text: str) -> list[float]:
        ...

    def embed_texts(self, texts: list[str]) -> list[list[float]]:
        ...

    def dimension(self) -> int:
        ...
```

这和前端 TypeScript 里的 interface 类似：

```ts
interface EmbeddingProvider {
  embedText(text: string): number[]
  embedTexts(texts: string[]): number[][]
  dimension(): number
}
```

## 参考实现解释

### Docker Compose 解释

```yaml
services:
```

表示定义一组服务。这里先只有一个服务：`postgres`。

```yaml
image: pgvector/pgvector:pg16
```

表示使用已经内置 pgvector 扩展的 PostgreSQL 16 镜像。

```yaml
environment:
```

表示传给容器的环境变量。

```yaml
ports:
  - "5432:5432"
```

表示：

```text
宿主机 5432 -> 容器内 5432
```

你本机 Python 连接 `localhost:5432`，实际连到容器里的 PostgreSQL。

```yaml
volumes:
  - postgres_data:/var/lib/postgresql/data
```

这是数据持久化。容器删了，volume 还在，数据库数据不会丢。

```yaml
- ./sql/init.sql:/docker-entrypoint-initdb.d/init.sql
```

第一次初始化数据库时，Postgres 镜像会自动执行这个 SQL。

注意：

```text
init.sql 只在数据库 volume 第一次创建时自动执行。
如果你改了 init.sql，但 volume 已存在，它不会重新执行。
```

需要重建：

```bash
docker compose down -v
docker compose up -d
```

## SQL 解释

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

启用 pgvector 扩展。

```sql
CREATE TABLE IF NOT EXISTS document_chunks
```

如果表不存在就创建。

```sql
id TEXT PRIMARY KEY
```

直接复用 JSON chunk 里的 UUID 字符串。第六周先不引入 UUID 数据库类型，降低复杂度。

```sql
metadata JSONB
```

PostgreSQL 的 JSONB 是二进制 JSON，适合保存结构化 metadata。

```sql
embedding vector(384)
```

保存 384 维向量。

```sql
embedding_status TEXT NOT NULL DEFAULT 'pending'
```

默认状态是 `pending`。

状态流转：

```text
pending -> processing -> completed
                   -> failed
```

```sql
USING hnsw (embedding vector_cosine_ops)
```

创建 HNSW 近似向量索引。pgvector 官方文档说明 HNSW 是支持的索引类型，并且 cosine distance 对应 `vector_cosine_ops`。([GitHub][1])

第六周数据量小，不建 HNSW 也能跑。这里建出来是为了提前熟悉真实工程做法。

### `sentence_transformer_provider.py`

```python
from sentence_transformers import SentenceTransformer

from src.app.config import get_embedding_model
from src.app.services.embedding.base import EmbeddingProvider


class SentenceTransformerEmbeddingProvider(EmbeddingProvider):
    def __init__(self, model_name: str | None = None) -> None:
        self.model_name = model_name or get_embedding_model()
        self.model = SentenceTransformer(self.model_name)

    def embed_text(self, text: str) -> list[float]:
        embeddings = self.embed_texts([text])
        return embeddings[0]

    def embed_texts(self, texts: list[str]) -> list[list[float]]:
        vectors = self.model.encode(
            texts,
            normalize_embeddings=True,
        )
        return vectors.tolist()

    def dimension(self) -> int:
        dimension = self.model.get_sentence_embedding_dimension()
        if dimension is None:
            raise RuntimeError("无法获取 embedding 模型维度")
        return int(dimension)
```

## 逐行解释

```python
from abc import ABC, abstractmethod
```

`ABC` 是 Abstract Base Class，抽象基类。

`@abstractmethod` 表示子类必须实现这个方法。

类比 TypeScript：

```ts
interface EmbeddingProvider {
  embedText(text: string): number[]
}
```

Python 没有完全一样的 interface，但 `ABC + abstractmethod` 可以表达类似约束。

---

```python
def embed_text(self, text: str) -> list[float]:
```

`list[float]` 表示返回一个浮点数列表。

例如：

```python
[0.01, -0.03, 0.18]
```

---

```python
def embed_texts(self, texts: list[str]) -> list[list[float]]:
```

`texts` 是多个字符串。

返回多个向量：

```python
[
    [0.01, -0.03],
    [0.11, 0.08],
]
```

---

```python
self.model = SentenceTransformer(self.model_name)
```

第一次运行会下载模型，所以会慢。

sentence-transformers 文档里说明 `SentenceTransformer` 支持 `cache_folder`，也可以通过 `SENTENCE_TRANSFORMERS_HOME` 环境变量指定模型缓存路径。([sbert.net][4])

---

```python
vectors = self.model.encode(texts, normalize_embeddings=True)
```

`encode()` 是把文本转成 embedding 的核心方法。官方文档显示 `encode()` 支持字符串或字符串列表，并且有 `batch_size`、`convert_to_numpy`、`normalize_embeddings` 等参数。([sbert.net][4])

```python
normalize_embeddings=True
```

表示把向量归一化成长度为 1。官方文档也明确说明这个参数会让返回向量长度为 1。([sbert.net][4])

归一化后：

```text
cosine similarity 和 dot product 更容易对应。
```

---

```python
return vectors.tolist()
```

`encode()` 默认返回 numpy array。官方文档说明默认返回 2D numpy array，形状是 `[num_inputs, output_dimension]`。([sbert.net][4])

`tolist()` 把 numpy array 转成普通 Python list。

原因：

```text
Pydantic、JSON、psycopg 参数传递时，普通 list 更直观。
```

```python
import json

from pgvector.psycopg import register_vector

from src.app.db import get_connection


class PgVectorStore:

                            %(chunk_index)s,
                            %(heading)s,
                            %(content)s,
                            %(char_count)s,
                            %(estimated_tokens)s,
                            %(metadata)s::jsonb,
                            NOW()
                        )
                        ON CONFLICT (id) DO UPDATE SET
                            document_id = EXCLUDED.document_id,
                            chunk_index = EXCLUDED.chunk_index,
                            heading = EXCLUDED.heading,
                            content = EXCLUDED.content,
                            char_count = EXCLUDED.char_count,
                            estimated_tokens = EXCLUDED.estimated_tokens,
                            metadata = EXCLUDED.metadata,
                            updated_at = NOW()
                        """,
                        {
                            "id": chunk["id"],
                            "document_id": chunk["document_id"],
                            "chunk_index": chunk["chunk_index"],
                            "heading": chunk.get("heading"),
                            "content": chunk["content"],
                            "char_count": chunk["char_count"],
                            "estimated_tokens": chunk["estimated_tokens"],
                            "metadata": json.dumps(chunk.get("metadata") or {}, ensure_ascii=False),
                        },
                    )
                    synced_count += 1

            conn.commit()

        return synced_count

    def list_chunks_by_document(self, document_id: str) -> list[dict[str, Any]]:
        with get_connection() as conn:
            self._register_vector(conn)

            with conn.cursor() as cur:
                cur.execute(
                    """
                    SELECT *
                    FROM document_chunks
                    WHERE document_id = %(document_id)s
                    ORDER BY chunk_index ASC
                    """,
                    {"document_id": document_id},
                )
                return list(cur.fetchall())

    def list_pending_chunks(self, limit: int = 100) -> list[dict[str, Any]]:
        with get_connection() as conn:
            self._register_vector(conn)

            with conn.cursor() as cur:
                cur.execute(
                    """
                    SELECT *
                    FROM document_chunks
                    WHERE embedding_status = 'pending'
                    ORDER BY created_at ASC
                    LIMIT %(limit)s
                    """,
                    {"limit": limit},
                )
                return list(cur.fetchall())

    def mark_chunks_processing(self, chunk_ids: list[str]) -> None:
        if not chunk_ids:
            return

        with get_connection() as conn:
            self._register_vector(conn)

            with conn.cursor() as cur:
                cur.execute(
                    """
                    UPDATE document_chunks
                    SET embedding_status = 'processing',
                        embedding_error = NULL,
                        embedding_updated_at = NOW(),
                        updated_at = NOW()
                    WHERE id = ANY(%(chunk_ids)s)
                    """,
                    {"chunk_ids": chunk_ids},
                )

            conn.commit()

    def update_chunk_embedding(
        self,
        chunk_id: str,
        embedding: list[float],
        embedding_model: str,
    ) -> None:
        with get_connection() as conn:
            self._register_vector(conn)

            with conn.cursor() as cur:
                cur.execute(
                    """
                    UPDATE document_chunks
                    SET embedding = %(embedding)s,
                        embedding_model = %(embedding_model)s,
                        embedding_status = 'completed',
                        embedding_error = NULL,
                        embedding_updated_at = NOW(),
                        updated_at = NOW()
                    WHERE id = %(chunk_id)s
                    """,
                    {
                        "chunk_id": chunk_id,
                        "embedding": embedding,
                        "embedding_model": embedding_model,
                    },
                )

            conn.commit()

    def mark_chunk_failed(self, chunk_id: str, error_message: str) -> None:
        with get_connection() as conn:
            with conn.cursor() as cur:
                cur.execute(
                    """
                    UPDATE document_chunks
                    SET embedding_status = 'failed',
                        embedding_error = %(error_message)s,
                        embedding_updated_at = NOW(),
                        updated_at = NOW()
                    WHERE id = %(chunk_id)s
                    """,
                    {
                        "chunk_id": chunk_id,
                        "error_message": error_message,
                    },
                )

            conn.commit()

    def get_document_embedding_status(self, document_id: str) -> list[dict[str, Any]]:
        with get_connection() as conn:
            with conn.cursor() as cur:
                cur.execute(
                    """
                    SELECT
                        id AS chunk_id,
                        chunk_index,
                        embedding_status,
                        embedding_model,
                        embedding_error,
                        embedding_updated_at
                    FROM document_chunks
                    WHERE document_id = %(document_id)s
                    ORDER BY chunk_index ASC
                    """,
                    {"document_id": document_id},
                )
                return list(cur.fetchall())

    def search_similar_chunks(
        self,
        query_embedding: list[float],
        top_k: int = 5,
    ) -> list[dict[str, Any]]:
        with get_connection() as conn:
            self._register_vector(conn)

            with conn.cursor() as cur:
                cur.execute(
                    """
                    SELECT
                        id AS chunk_id,
                        document_id,
                        chunk_index,
                        heading,
                        content,
                        embedding <=> %(query_embedding)s AS distance,
                        1 - (embedding <=> %(query_embedding)s) AS score
                    FROM document_chunks
                    WHERE embedding IS NOT NULL
                      AND embedding_status = 'completed'
                    ORDER BY embedding <=> %(query_embedding)s
                    LIMIT %(top_k)s
                    """,
                    {
                        "query_embedding": query_embedding,
                        "top_k": top_k,
                    },
                )
                return list(cur.fetchall())
```

## `embedding_service.py` 参考实现

```python
from typing import Any

from src.app.config import (
    get_embedding_batch_size,
    get_embedding_dimension,
    get_embedding_model,
)
from src.app.document_store import load_document_chunks
from src.app.exceptions import AppError
from src.app.services.embedding.sentence_transformer_provider import (
    SentenceTransformerEmbeddingProvider,
)
from src.app.services.vector_store import PgVectorStore


class EmbeddingService:
    def __init__(self) -> None:
        self.provider = SentenceTransformerEmbeddingProvider()
        self.vector_store = PgVectorStore()

    def test_embedding(self, text: str) -> dict[str, Any]:
        vector = self.provider.embed_text(text)
        self._validate_dimension(vector)

        vector_norm = sum(value * value for value in vector) ** 0.5

        return {
            "text": text,
            "embedding_model": get_embedding_model(),
            "dimension": len(vector),
            "vector_preview": [round(value, 4) for value in vector[:5]],
            "vector_norm": round(vector_norm, 4),
        }

    def embed_document(self, document_id: str) -> dict[str, Any]:
        all_chunks = load_document_chunks()
        self.vector_store.sync_chunks_from_json(all_chunks)

        chunks = self.vector_store.list_chunks_by_document(document_id)

        if not chunks:
            raise AppError(
                code="DOCUMENT_CHUNKS_NOT_FOUND",
                message="该文档没有可向量化的 chunks",
                detail=f"document_id={document_id}",
                status_code=404,
            )

        return self._embed_chunks(
            chunks=chunks,
            document_id=document_id,
        )

    def embed_all_pending(self) -> dict[str, Any]:
        all_chunks = load_document_chunks()
        self.vector_store.sync_chunks_from_json(all_chunks)

        limit = get_embedding_batch_size()
        chunks = self.vector_store.list_pending_chunks(limit=limit)

        result = self._embed_chunks(chunks=chunks)

        return {
            "total_chunks": result["total_chunks"],
            "embedded_chunks": result["embedded_chunks"],
            "failed_chunks": result["failed_chunks"],
            "embedding_model": result["embedding_model"],
            "status": result["status"],
        }

    def get_document_embedding_status(self, document_id: str) -> dict[str, Any]:
        items = self.vector_store.get_document_embedding_status(document_id)

        return {
            "document_id": document_id,
            "items": [
                {
                    **item,
                    "embedding_updated_at": (
                        item["embedding_updated_at"].isoformat()
                        if item.get("embedding_updated_at")
                        else None
                    ),
                }
                for item in items
            ],
        }

    def search_debug(self, query: str, top_k: int = 5) -> dict[str, Any]:
        query_embedding = self.provider.embed_text(query)
        self._validate_dimension(query_embedding)

        items = self.vector_store.search_similar_chunks(
            query_embedding=query_embedding,
            top_k=top_k,
        )

        return {
            "query": query,
            "embedding_model": get_embedding_model(),
            "top_k": top_k,
            "items": [
                {
                    **item,
                    "distance": round(float(item["distance"]), 4),
                    "score": round(float(item["score"]), 4),
                }
                for item in items
            ],
        }

    def _embed_chunks(
        self,
        chunks: list[dict[str, Any]],
        document_id: str | None = None,
    ) -> dict[str, Any]:
        total_chunks = len(chunks)
        embedded_chunks = 0
        failed_chunks = 0

        if not chunks:
            return {
                "document_id": document_id,
                "total_chunks": 0,
                "embedded_chunks": 0,
                "failed_chunks": 0,
                "embedding_model": get_embedding_model(),
                "status": "completed",
            }

        chunk_ids = [chunk["id"] for chunk in chunks]
        texts = [chunk["content"] for chunk in chunks]

        self.vector_store.mark_chunks_processing(chunk_ids)

        try:
            embeddings = self.provider.embed_texts(texts)

            for chunk, embedding in zip(chunks, embeddings):
                self._validate_dimension(embedding)

                self.vector_store.update_chunk_embedding(
                    chunk_id=chunk["id"],
                    embedding=embedding,
                    embedding_model=get_embedding_model(),
                )
                embedded_chunks += 1

        except Exception as exc:
            failed_chunks = total_chunks - embedded_chunks

            for chunk in chunks[embedded_chunks:]:
                self.vector_store.mark_chunk_failed(
                    chunk_id=chunk["id"],
                    error_message=str(exc),
                )

        status = "completed" if failed_chunks == 0 else "failed"

        return {
            "document_id": document_id,
            "total_chunks": total_chunks,
            "embedded_chunks": embedded_chunks,
            "failed_chunks": failed_chunks,
            "embedding_model": get_embedding_model(),
            "status": status,
        }

    def _validate_dimension(self, vector: list[float]) -> None:
        expected_dimension = get_embedding_dimension()
        actual_dimension = len(vector)

        if actual_dimension != expected_dimension:
            raise AppError(
                code="EMBEDDING_DIMENSION_MISMATCH",
                message="Embedding 维度和配置不一致",
                detail=f"expected={expected_dimension}, actual={actual_dimension}",
                status_code=500,
            )
```

## SQL 解释

核心 SQL：

```sql
SELECT
    id AS chunk_id,
    document_id,
    chunk_index,
    heading,
    content,
    embedding <=> %(query_embedding)s AS distance,
    1 - (embedding <=> %(query_embedding)s) AS score
FROM document_chunks
WHERE embedding IS NOT NULL
  AND embedding_status = 'completed'
ORDER BY embedding <=> %(query_embedding)s
LIMIT %(top_k)s
```

逐句解释：

```sql
embedding <=> query_embedding
```

计算 cosine distance。

```sql
ORDER BY embedding <=> query_embedding
```

按距离从小到大排序。

```sql
LIMIT top_k
```

只取前 k 条。

```sql
1 - distance AS score
```

把 distance 转成更符合直觉的 score。

# 本周必须掌握的 AI 工程概念

```text
1. Embedding 是文本的向量表示。
2. Chat Model 负责生成回答，Embedding Model 负责生成向量。
3. Chunk 要先 embedding，才能做语义检索。
4. vector(n) 的 n 必须等于模型输出维度。
5. cosine distance 越小越相似。
6. score = 1 - distance 更符合直觉。
7. pgvector 把向量能力放进 PostgreSQL。
8. embedding_status 用于任务状态管理。
9. batch embedding 比逐条 embedding 高效。
10. search-debug 是检索调试，不是 RAG。
11. RAG 必须等到 Week 7：检索结果进入 prompt，再调用 LLM 回答。
```

## 3 分钟版本

```text
我把 Week 6 拆成三层：

第一层是模型层：
EmbeddingProvider 定义 embed_text、embed_texts 和 dimension。当前实现用 sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2，支持中文和英文，输出 384 维向量。

第二层是存储层：
PgVectorStore 使用 PostgreSQL + pgvector。document_chunks 表里保存 chunk 的业务字段、metadata JSONB、embedding vector(384)、embedding_model、embedding_status、embedding_error 和 embedding_updated_at。这样可以追踪每个 chunk 的向量化状态。

第三层是编排层：
EmbeddingService 负责同步 JSON chunks 到 PostgreSQL，批量生成 embedding，校验维度，更新状态，并提供 search-debug。search-debug 会把 query 转成向量，然后用 pgvector 的 cosine distance 查询 top_k chunks。

我没有在第六周做 RAG Answer，因为 RAG 是 query -> retrieve -> build prompt -> LLM answer -> sources。第六周只做 retrieve 前置能力，确保向量化和检索质量可调试。
```

## 5 分钟版本核心结构

```text
1. 背景：
   Week 5 已经完成 document -> text -> chunk，但 chunk 只是文本，不能做语义搜索。

2. 技术选型：
   PostgreSQL + pgvector 管业务字段和向量；
   sentence-transformers 负责本地 embedding；
   psycopg 负责 Python 连接 PostgreSQL。

3. 架构：
   routes.py -> EmbeddingService -> EmbeddingProvider + PgVectorStore -> PostgreSQL。

4. 数据库：
   document_chunks 表保留 JSON chunk id，新增 embedding vector(384)、status、model、error、updated_at。

6. 批量处理：
   不一个 chunk 一个模型调用，而是 texts list 批量 encode，提高吞吐。

7. 检索调试：
   search-debug 使用 query embedding 和 `<=>` cosine distance 返回 top_k chunks，同时返回 distance 和 score。

8. 边界：
   第六周不做 RAG 问答，不做 Agent，不做 MCP，只打通 embedding + vector store。
```
