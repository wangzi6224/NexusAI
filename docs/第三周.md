# Day 1：理解文件上传与 multipart/form-data

## 1.1 为什么文件上传不是 JSON？

你之前 `/chat` 是 JSON：

```json
{
  "message": "什么是 RAG？"
}
```

JSON 适合结构化文本数据，不适合直接传文件。

文件上传一般使用：

```http
Content-Type: multipart/form-data
```

它的意思是：

```text
一个 HTTP 请求体里，可以包含多个 part：
- 普通字段
- 文件字段
- 每个 part 有自己的 headers
- 文件内容可以是二进制 bytes
```

前端类比：

```ts
const formData = new FormData()
formData.append("file", file)

fetch("/documents/upload", {
  method: "POST",
  body: formData,
})
```

注意：这里不能手动写：

```ts
headers: {
  "Content-Type": "application/json"
}
```

因为浏览器需要自动生成 multipart boundary。

---

## 1.2 FastAPI 中的 UploadFile 是什么？

FastAPI 上传文件常用：

```python
from fastapi import UploadFile, File

async def upload_document(file: UploadFile = File(...)):
    ...
```

### `UploadFile`

它代表上传上来的文件对象。

常用属性：

```python
file.filename
file.content_type
await file.read()
```

| 属性 / 方法        | 说明                                       |
| -------------- | ---------------------------------------- |
| `filename`     | 用户上传时的原始文件名，例如 `frontend-style-guide.md` |
| `content_type` | 浏览器或客户端声明的 MIME 类型，例如 `text/markdown`    |
| `read()`       | 读取文件内容，返回 `bytes`                        |

### `File(...)`

`File(...)` 告诉 FastAPI：

```text
这个参数不是 query
不是 JSON body
而是 multipart/form-data 里的文件字段
```

`...` 表示必填。

---

## 1.3 为什么 `await file.read()` 是 async？

文件读取可能涉及 IO：

```text
客户端上传
  ↓
服务器临时缓存
  ↓
应用读取 bytes
```

IO 操作可能慢。FastAPI 支持 async，是为了在等待 IO 时不阻塞整个服务。

你的当前项目很多路由是同步函数，例如 `/chat` 是普通 `def`。这没问题。第三周上传接口建议用 `async def`，因为 `UploadFile.read()` 是异步方法。

## 2.3 字段解释

| 字段               | 作用           | 为什么重要                     |
| ---------------- | ------------ | ------------------------- |
| `document_id`    | 文档唯一 ID      | 后续查详情、查 chunks、删除、重建索引都靠它 |
| `filename`       | 原始文件名        | 前端展示、用户识别来源               |
| `file_type`      | `md` / `txt` | 决定走哪种解析和切分策略              |
| `status`         | 处理状态         | 前端展示是否成功                  |
| `content_length` | 文本长度         | 调试文档是否正常读取                |
| `chunk_count`    | chunk 数量     | 评估切分结果                    |
| `created_at`     | ISO 时间字符串    | 排序、审计、调试                  |
| `chunk_index`    | chunk 顺序     | 后续还原上下文、展示来源              |
| `heading`        | Markdown 标题  | 保留语义结构                    |
| `metadata`       | 附加信息         | 后续检索、引用、前端展示依据            |

为什么 `document_id` 用 UUID 字符串？

```text
1. 不依赖数据库自增 ID
2. 本地 JSON 阶段也能生成唯一 ID
3. 未来迁移数据库也能继续使用
4. 前端、日志、chunk 都能稳定引用
```

为什么 `created_at` 用 ISO 字符串？

```python
datetime.now().isoformat()
```

返回类似：

```text
2026-04-29T15:30:12.123456
```

优点：

```text
1. JSON 原生支持字符串
2. 前端容易解析
3. 日志可读
4. 后续数据库迁移容易
```

## 2.5 参考实现

```python
from typing import Any, Literal
from pydantic import BaseModel, Field


DocumentStatus = Literal["completed", "failed"]


class DocumentUploadResponse(BaseModel):
    document_id: str = Field(..., description="文档唯一 ID")
    filename: str = Field(..., description="上传文件名")
    file_type: str = Field(..., description="文件类型：md 或 txt")
    status: DocumentStatus = Field(..., description="处理状态")
    content_length: int = Field(..., description="文本字符长度")
    chunk_count: int = Field(..., description="切分后的 chunk 数量")
    created_at: str = Field(..., description="创建时间，ISO 字符串")


class DocumentListItem(BaseModel):
    document_id: str
    filename: str
    file_type: str
    status: DocumentStatus
    chunk_count: int
    created_at: str


class DocumentDetail(BaseModel):
    document_id: str
    filename: str
    file_type: str
    status: DocumentStatus
    content_length: int
    chunk_count: int
    created_at: str
    metadata: dict[str, Any] = Field(default_factory=dict)


class DocumentChunkItem(BaseModel):
    chunk_id: str
    document_id: str
    chunk_index: int
    content: str
    char_count: int
    estimated_tokens: int
    heading: str | None = None
    metadata: dict[str, Any] = Field(default_factory=dict)
```

逐行解释重点：

```python
from typing import Any, Literal
```

`Literal` 用来限制字段只能是指定字符串。`Any` 用来表示 metadata 暂时允许任意 JSON 值。

```python
DocumentStatus = Literal["completed", "failed"]
```

定义状态类型。第三周简化，只保留完成和失败。

```python
Field(..., description="...")
```

`...` 表示必填，`description` 会进入 OpenAPI 文档。

```python
metadata: dict[str, Any] = Field(default_factory=dict)
```

不要写：

```python
metadata: dict[str, Any] = {}
```

因为 `{}` 是可变对象，多个实例可能共享同一个默认值。`default_factory=dict` 每次创建一个新字典。


## 3.5 参考实现

```python
import json
from pathlib import Path
from typing import Any


DATA_DIR = Path("data")
DOCUMENTS_DIR = DATA_DIR / "documents"
CHUNKS_DIR = DATA_DIR / "chunks"


def ensure_store_dirs() -> None:
    DOCUMENTS_DIR.mkdir(parents=True, exist_ok=True)
    CHUNKS_DIR.mkdir(parents=True, exist_ok=True)


def save_document(document: dict[str, Any]) -> None:
    ensure_store_dirs()

    document_id = document["document_id"]
    file_path = DOCUMENTS_DIR / f"{document_id}.json"

    file_path.write_text(
        json.dumps(document, ensure_ascii=False, indent=2),
        encoding="utf-8",
    )


def save_chunks(document_id: str, chunks: list[dict[str, Any]]) -> None:
    ensure_store_dirs()

    file_path = CHUNKS_DIR / f"{document_id}.json"

    file_path.write_text(
        json.dumps(chunks, ensure_ascii=False, indent=2),
        encoding="utf-8",
    )


def list_documents() -> list[dict[str, Any]]:
    ensure_store_dirs()

    documents: list[dict[str, Any]] = []

    for file_path in DOCUMENTS_DIR.glob("*.json"):
        content = file_path.read_text(encoding="utf-8")
        documents.append(json.loads(content))

    return sorted(documents, key=lambda item: item["created_at"], reverse=True)


def get_document(document_id: str) -> dict[str, Any] | None:
    ensure_store_dirs()

    file_path = DOCUMENTS_DIR / f"{document_id}.json"

    if not file_path.exists():
        return None

    return json.loads(file_path.read_text(encoding="utf-8"))


def get_chunks(document_id: str) -> list[dict[str, Any]]:
    ensure_store_dirs()

    file_path = CHUNKS_DIR / f"{document_id}.json"

    if not file_path.exists():
        return []

    return json.loads(file_path.read_text(encoding="utf-8"))
```

逐行解释重点：

```python
from pathlib import Path
```

`Path` 是 Python 标准库里处理路径的对象，比字符串拼路径更安全。

```python
DATA_DIR = Path("data")
```

项目根目录下的 `data` 目录。

```python
mkdir(parents=True, exist_ok=True)
```

含义：

```text
parents=True：父目录不存在时一起创建
exist_ok=True：目录已存在时不报错
```

```python
json.dumps(document, ensure_ascii=False, indent=2)
```

`ensure_ascii=False`：中文不转义成 `\u4e2d\u6587`。
`indent=2`：格式化 JSON，方便你直接打开看。

```python
sorted(..., key=lambda item: item["created_at"], reverse=True)
```

按创建时间倒序，最新上传的文档排最前面。

---

## 4.2 为什么文件读出来是 bytes？

HTTP 上传的是二进制流。服务器先拿到的是：

```python
bytes
```

比如：

```python
b'# Frontend Guide\n\nhello'
```

模型和 splitter 需要的是字符串：

```python
"# Frontend Guide\n\nhello"
```

所以要：

```python
content.decode("utf-8")
```

---

## 4.3 如果不是 UTF-8 会怎样？

如果文件实际是 GBK，但你用 UTF-8 解码，可能抛：

```python
UnicodeDecodeError
```

```python
SUPPORTED_FILE_TYPES = {
    ".md": "md",
    ".txt": "txt",

def validate_document_file(filename: str) -> str:

def validate_file_size(

    try:
```

# Day 5：实现 Chunk Splitter

## 5.1 为什么需要 chunk？

大文档不能直接塞给模型，原因有四个：

```text
1. 模型上下文窗口有限
2. 大文档成本高
3. 检索时需要定位小片段
4. 回答时需要引用来源
```

RAG 的核心不是“把文档存起来”，而是：

```text
把文档切成适合检索的小语义单元
```

这些小语义单元就是 chunk。

---

## 5.2 chunk 太大有什么问题？

```text
1. 一个 chunk 混入多个主题
2. 检索命中后上下文噪声大
3. 模型回答不够聚焦
4. token 成本高
```

例如一个 chunk 同时包含：

```text
组件命名
Hooks 命名
API 分页
错误码
```

用户问“组件怎么命名”，检索回来一大坨，模型容易跑偏。

---

## 5.3 chunk 太小有什么问题？

```text
1. 语义不完整
2. 标题和正文分离
3. 检索命中碎片后模型不知道上下文
4. 召回数量变多
```

例如只切出：

```text
PascalCase
```

模型不知道这是组件命名规则，还是类名规则。

---

## 5.4 overlap 是什么？

overlap 是相邻 chunk 之间保留一小段重复内容。

例如：

```text
chunk 1: A B C D E
chunk 2: D E F G H
```

`D E` 就是 overlap。

为什么需要？

```text
文档切分可能刚好把一个语义拆断。
overlap 可以降低语义断裂风险。
```

---

## 5.5 Markdown 标题为什么重要？

Markdown 是半结构化文本。

标题天然表达层级：

```md
# 前端规范

## 组件命名规范

组件必须使用 PascalCase。
```

如果 chunk 只保存：

```text
组件必须使用 PascalCase。
```

就丢失了语义背景。

如果 chunk 保存：

```json
{
  "heading": "组件命名规范",
  "content": "组件必须使用 PascalCase。"
}
```

后续 RAG 可以告诉用户：

```text
来源：frontend-style-guide.md / 组件命名规范
```

这对引用、调试、前端展示都非常重要。

---

## 5.6 第三周 splitter 策略

第一版策略：

### `.txt`

```text
按段落切
段落太长时按字符长度切
支持 overlap
```

### `.md`

```text
识别 #、##、### 标题
尽量把标题放进 chunk.heading
标题下内容按段落切
太长时按字符长度切
```

### 暂定参数

```python
chunk_size = 500
overlap = 80
```

这是字符数，不是真实 token 数。第三周先用字符数简化。

## 5.9 参考实现：document_splitter.py

```python
import re
import uuid
from typing import Any


DEFAULT_CHUNK_SIZE = 500
DEFAULT_OVERLAP = 80


def estimate_token_count(text: str) -> int:
    return max(1, len(text) // 4)


def split_long_text(
    text: str,
    chunk_size: int = DEFAULT_CHUNK_SIZE,
    overlap: int = DEFAULT_OVERLAP,
) -> list[str]:
    text = text.strip()

    if not text:
        return []

    if len(text) <= chunk_size:
        return [text]

    chunks: list[str] = []
    start = 0

    while start < len(text):
        end = start + chunk_size
        chunk = text[start:end].strip()

        if chunk:
            chunks.append(chunk)

        if end >= len(text):
            break

        start = max(0, end - overlap)

    return chunks


def split_text_into_chunks(
    text: str,
    chunk_size: int = DEFAULT_CHUNK_SIZE,
    overlap: int = DEFAULT_OVERLAP,
) -> list[dict[str, Any]]:
    paragraphs = [item.strip() for item in re.split(r"\n\s*\n", text) if item.strip()]

    raw_chunks: list[str] = []
    buffer = ""

    for paragraph in paragraphs:
        if len(paragraph) > chunk_size:
            if buffer.strip():
                raw_chunks.append(buffer.strip())
                buffer = ""

            raw_chunks.extend(split_long_text(paragraph, chunk_size, overlap))
            continue

        next_buffer = f"{buffer}\n\n{paragraph}".strip()

        if len(next_buffer) <= chunk_size:
            buffer = next_buffer
        else:
            if buffer.strip():
                raw_chunks.append(buffer.strip())
            buffer = paragraph

    if buffer.strip():
        raw_chunks.append(buffer.strip())

    return [
        build_chunk_dict(
            content=chunk,
            chunk_index=index,
            heading=None,
            metadata={"splitter": "paragraph"},
        )
        for index, chunk in enumerate(raw_chunks)
    ]


def split_markdown_into_chunks(
    text: str,
    chunk_size: int = DEFAULT_CHUNK_SIZE,
    overlap: int = DEFAULT_OVERLAP,
) -> list[dict[str, Any]]:
    sections = parse_markdown_sections(text)

    raw_chunks: list[dict[str, Any]] = []

    for section in sections:
        heading = section["heading"]
        content = section["content"].strip()

        if not content:
            continue

        section_chunks = split_text_into_chunks(
            text=content,
            chunk_size=chunk_size,
            overlap=overlap,
        )

        for section_chunk in section_chunks:
            raw_chunks.append(
                {
                    "content": section_chunk["content"],
                    "heading": heading,
                    "metadata": {
                        "splitter": "markdown",
                        "heading": heading,
                    },
                }
            )

    return [
        build_chunk_dict(
            content=item["content"],
            chunk_index=index,
            heading=item["heading"],
            metadata=item["metadata"],
        )
        for index, item in enumerate(raw_chunks)
    ]


def parse_markdown_sections(text: str) -> list[dict[str, str | None]]:
    lines = text.splitlines()

    sections: list[dict[str, str | None]] = []
    current_heading: str | None = None
    current_lines: list[str] = []

    heading_pattern = re.compile(r"^(#{1,3})\s+(.+)$")

    for line in lines:
        match = heading_pattern.match(line)

        if match:
            if current_lines:
                sections.append(
                    {
                        "heading": current_heading,
                        "content": "\n".join(current_lines),
                    }
                )

            current_heading = match.group(2).strip()
            current_lines = [line]
        else:
            current_lines.append(line)

    if current_lines:
        sections.append(
            {
                "heading": current_heading,
                "content": "\n".join(current_lines),
            }
        )

    return sections


def build_chunk_dict(
    content: str,
    chunk_index: int,
    heading: str | None,
    metadata: dict[str, Any],
) -> dict[str, Any]:
    return {
        "chunk_id": str(uuid.uuid4()),
        "chunk_index": chunk_index,
        "content": content,
        "char_count": len(content),
        "estimated_tokens": estimate_token_count(content),
        "heading": heading,
        "metadata": metadata,
    }


def split_document_content(file_type: str, text: str) -> list[dict[str, Any]]:
    if file_type == "md":
        return split_markdown_into_chunks(text)

    if file_type == "txt":
        return split_text_into_chunks(text)

    return split_text_into_chunks(text)
```

---

## 5.10 关键代码逐段解释

### `estimate_token_count`

```python

```

真实 token 统计要用 tokenizer。第三周先粗略估算：

```text
英文大约 4 个字符一个 token
中文不完全准确
但足够做调试字段
```

`max(1, ...)` 是防止非常短文本估算成 0。

---

### `split_long_text`

```python
if len(text) <= chunk_size:
    return [text]
```

短文本不用切。

```python

```

从第 0 个字符开始切。

```python
end = start + chunk_size
```

结束位置是开始位置加 chunk size。

```python
start = max(0, end - overlap)
```

下一段从 `end - overlap` 开始，实现重叠。

例如：

```text
chunk_size = 500
overlap = 80

第一段：0 - 500
第二段：420 - 920
```

---

### `split_text_into_chunks`

```python
paragraphs = [item.strip() for item in re.split(r"\n\s*\n", text) if item.strip()]
```

按空行切段落。

```text

\s*     中间可能有空格
```

```python
import uuid

from fastapi import UploadFile

from src.app.document_store import (
    get_chunks,
    get_document,
    list_documents,
    save_chunks,
    save_document,

from src.app.logger import get_logger
from src.app.schemas import (
    DocumentChunkItem,
    DocumentDetail,
    DocumentListItem,
    DocumentUploadResponse,

from src.app.services.document_loader import (

    validate_document_file,
    validate_file_size,

from src.app.services.document_splitter import split_document_content


logger = get_logger()


async def upload_document_file(file: UploadFile) -> DocumentUploadResponse:
    filename = file.filename or "unknown"

    file_type = validate_document_file(filename)

    content_bytes = await file.read()
    validate_file_size(content_bytes)

    text = decode_document_content(content_bytes, filename)

    document_id = str(uuid.uuid4())
    created_at = datetime.now().isoformat()

    chunks = split_document_content(file_type=file_type, text=text)

    for chunk in chunks:
        chunk["document_id"] = document_id
        chunk["metadata"] = {
            **chunk.get("metadata", {}),
            "filename": filename,
            "file_type": file_type,
            "document_id": document_id,
        }

    document = {
        "document_id": document_id,
        "filename": filename,
        "file_type": file_type,
        "status": "completed",
        "content_length": len(text),
        "chunk_count": len(chunks),
        "created_at": created_at,
        "metadata": {
            "source": "upload",
            "content_type": file.content_type,
            "size_bytes": len(content_bytes),
        },
    }

    save_document(document)
    save_chunks(document_id, chunks)

    logger.info(
        "文档上传成功: filename=%s size=%s chunk_count=%s",
        filename,
        len(content_bytes),
        len(chunks),
    )

    return DocumentUploadResponse(**document)


def list_document_items() -> list[DocumentListItem]:
    return [DocumentListItem(**item) for item in list_documents()]

    document = get_document(document_id)

    if document is None:
        raise DocumentError(
            message="文档不存在",
            detail=f"document_id={document_id}",
            status_code=404,
        )

    return DocumentDetail(**document)

    document = get_document(document_id)

    if document is None:
        raise DocumentError(

            detail=f"document_id={document_id}",
```

# 12. 本周必须掌握的概念

## 12.1 UploadFile

`UploadFile` 是 FastAPI 对上传文件的封装。

你要会解释：

```text
它不是字符串，也不是 JSON。
它代表 multipart/form-data 中的文件 part。
```

---

## 12.2 multipart/form-data

你要会解释：

```text
multipart/form-data 是 HTTP 上传文件常用格式。
一个请求体里可以包含多个字段，每个字段可以是文本，也可以是文件。
```

前端类比：

```ts
const formData = new FormData()
formData.append("file", file)
```

---

## 12.3 bytes 与 str

你要会解释：

```text
网络上传来的文件首先是 bytes。
文本处理需要 str。
bytes 需要用 decode("utf-8") 转成字符串。
```

---

## 12.4 chunk

你要会解释：

```text
chunk 是从大文档中切出来的小文本片段。
RAG 后续不是直接检索整篇文档，而是检索最相关的 chunk。
```

---

## 12.5 overlap

你要会解释：

```text
overlap 是相邻 chunk 之间保留一段重复文本。
它可以降低切分边界导致的语义断裂。
```

---

## 12.6 metadata

你要会解释：

```text
metadata 不是随便附加字段。
它用于记录来源、标题、文件类型、切分策略等信息。
后续检索结果展示、引用来源、调试问题都依赖 metadata。
```

# 13. 第三周完成后的面试表达
