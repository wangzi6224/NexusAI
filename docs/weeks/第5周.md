# 第五周教程：文档上传 Document Upload + Chunk 切分

本教程基于我实际读取到的 GitHub 仓库 `wangzi6224/my_python_project` 当前代码状态编写，不是凭空设计。

我核对到的关键现状：

* `routes.py` 当前仍是集中式路由文件，已经有 `/chat`、`/chat/stream`、会话、消息、context-preview、summary 等接口；其中 `ContextBuilder` 被 import 但没有被使用，建议删除。
* `context_builder.py` 当前已经实现 system prompt、summary、recent messages、max_context_chars、estimated_tokens、context preview；但 import 了未使用的 `ContextPreviewResponse`，建议删除。
* `conversation_store.py` 当前已经用 JSON 存储 `conversations.json` 和 `messages.json`，并且 `create_conversation` 已初始化 `summarized_message_count` 和 `summary_updated_at`。
* `conversation_service.py` 当前普通消息和流式消息都会先保存 user message，再构造上下文，再调用模型，再保存 assistant message；assistant 保存后才尝试自动 summary。
* `stream_conversation_message` 里 `message_start` 在模型流式输出前发送，自动 summary 在 assistant message 保存后才触发，因此 `message_start` 不会被 summary 阻塞。
* `schemas.py` 当前仍是集中式 Schema 文件，已有 Chat、Conversation、Message、ContextPreview、Summary 相关模型。第五周继续放这里，暂不拆分。
* `paths.py` 当前 `DATA_DIR = BASE_ABS_DIR / "data"`，也就是说实际数据目录是 `src/app/data`，不是仓库根目录的 `data/`。第五周建议沿用当前 `DATA_DIR`，新增 `UPLOAD_DIR = DATA_DIR / "uploads"`。
* `pyproject.toml` 当前依赖里没有 `python-multipart`，上传文件接口必须补这个依赖。
* 你的总体学习计划里 Week 5 的主线就是“上传文档 -> 解析文本 -> 切分 chunk -> 保存 document 和 chunk”，并明确第一版只做 `.md` / `.txt`，暂不做 PDF。

## 4. 为什么 AI 知识库必须先有 Document / Chunk 层

### 4.1 Document 是知识库的一级资源

`Document` 不是“上传文件记录”这么简单。

它代表知识库里的一份原始资料。

真实 AI 工程中，Document 解决这些问题：

```text
这份知识来自哪里？
原始文件叫什么？
文件保存在哪里？
解析是否成功？
什么时候上传？
后续是否需要重建索引？
失败原因是什么？
```

对应字段：

```text
id
filename
file_type
source_path
content
status
error_message
char_count
chunk_count
created_at
updated_at
```

类比前端 / Node.js：

```text
Document 类似 CMS 里的 Article / Asset / Resource
不是临时文件，而是业务资源。
```

---

### 4.2 Chunk 是 RAG 检索的最小语义单元

Chunk 不是普通字符串切片。

它是后续 Embedding 和 RAG 检索的最小单位。

后面 Week 6 会做：

```text
chunk.content -> embedding vector -> pgvector
```

Week 7 会做：

```text
用户问题 -> query embedding -> 检索相关 chunks -> 构造 RAG prompt
```

如果 Week 5 的 chunk 切得很差，后面 RAG 会直接变差。

例如：

```text
chunk 太短：信息不足，检索到了也回答不了
chunk 太长：主题混杂，embedding 表达模糊
chunk 不保留 heading：回答时很难解释来源
chunk 无 metadata：后续无法过滤、溯源、调试
```

---

### 4.3 Loader 是 AI 数据入口

`UploadFile` 不只是上传文件。

在 AI 应用里，它是：

```text
非结构化数据进入系统的入口。
```

Loader 负责把外部文件变成内部可处理的文本。

本周只做：

```text
.md / .txt -> utf-8 text
```

不做：

```text
PDF
OCR
Word
Excel
PPT
```

---

### 4.4 Splitter 是影响召回质量的核心模块

Splitter 不是工具函数。

它决定：

```text
后续 embedding 表达什么语义
后续 vector search 能不能召回正确内容
后续 RAG 回答有没有可靠上下文
```

第五周第一版策略：

```text
Markdown：
  优先按标题识别结构
  每个标题下内容作为 section
  section 太长时按段落拆
  段落仍然太长时按字符长度兜底拆
  chunk 保留 heading

txt：
  按段落拆
  段落太长时按字符长度兜底拆
  heading = None
```

---

### 4.5 为什么第五周不做 PDF

PDF 解析复杂度明显高于 `.md` / `.txt`。

它会引入：

```text
版面分析
字体编码
坐标
多栏阅读顺序
页眉页脚
表格
图片
OCR
扫描件
文本提取误差
```

这会严重偏离第五周主线。

第五周目标不是“做一个全格式文档解析器”，而是建立 RAG 前置管线：

```text
document -> text -> chunk
```

PDF 可以后续单独做专题。

## 5. Document、Chunk、Loader、Splitter、Service 核心概念

| 模块                  | 是什么          | 真实工程价值                 | Node / 前端类比                               |
| ------------------- | ------------ | ---------------------- | ----------------------------------------- |
| `Document`          | 文档一级资源       | 管理来源、状态、原文、统计信息        | CMS 的 Article / Asset                     |
| `DocumentChunk`     | 文档切分后的语义片段   | Embedding / RAG 检索最小单元 | 搜索索引里的 document segment                   |
| `DocumentLoader`    | 文件校验、保存、文本读取 | 控制外部数据入口安全             | Express multer upload middleware + parser |
| `ChunkSplitter`     | 文本切分策略模块     | 决定后续召回质量               | 文本预处理 pipeline                            |
| `DocumentService`   | 编排层          | 串起上传、解析、切分、保存          | NestJS Service / Application Service      |
| `document_store.py` | JSON 存储层     | 学习阶段可视化存储              | JSON 文件版 repository                       |

## 字段逐项解释

### `DocumentItem.id`

文档唯一 ID。

不要用 filename 当 ID，因为：

```text
文件名可能重复
文件名可能包含危险字符
文件名可能被用户改动
```

### `filename`

用户上传时的原始文件名，主要用于展示。

### `source_path`

服务端保存后的真实路径。

它和 `filename` 不同：

```text
filename: fastapi-guide.md
source_path: src/app/data/uploads/uuid_fastapi-guide.md
```

`source_path` 后续用于：

```text
排查上传文件
重建索引
重新解析
删除原始文件
```

### `file_type`

建议保存为：

```text
md
txt
```

不要只在每次使用时重新从 filename 猜。

因为后续文件名可能不可信，业务处理应该依赖系统校验后的结果。

### `status`

文档状态：

```text
pending
processing
completed
failed
```

真实系统中上传解析经常变成异步任务，状态字段是后续扩展基础。

### `error_message`

失败时记录原因。

例如：

```text
文件类型不支持
文件过大
文本解码失败
chunk 为空
```

### `content`

原始文本内容。

只在详情接口返回，不建议列表接口返回。

原因：

```text
列表接口可能很多文档
content 可能很长
影响响应性能
前端列表也不需要全文
```

### `chunk_index`

同一个 document 下的 chunk 顺序。

必须稳定连续：

```text
0, 1, 2, 3...
```

后续用于：

```text
调试
上下文重组
前端预览
引用排序
```

### `heading`

Markdown 标题。

它能保留文档结构，让 chunk 更可解释。

例如：

```text
heading = JSONResponse
content = JSONResponse 可以自定义异常返回结构...
```

比单独 content 更容易理解来源。

### `char_count`

字符数。

用于：

```text
检查 chunk 是否过长
控制切分策略
调试异常 chunk
```

### `estimated_tokens`

粗略 token 估算。

和第四周 ContextBuilder 的思想一致，但不要直接依赖 ContextBuilder。

因为：

```text
ContextBuilder 属于会话上下文
ChunkSplitter 属于文档处理
两个模块职责不同
```

### `metadata`

后续非常重要。

可以保存：

```json
{
  "split_method": "markdown_heading",
  "file_type": "md",
  "heading_level": 2
}
```

后续 RAG 可以用它做：

```text
过滤
溯源
展示
评测
调试
```

## 参考实现：`document_store.py`

```python
import json
import uuid
from datetime import datetime
from pathlib import Path
from typing import Any

from src.app.exceptions import AppError
from src.app.paths import DATA_DIR, DOCUMENTS_FILE, DOCUMENT_CHUNKS_FILE


def _now_iso() -> str:
    return datetime.now().isoformat()


def _ensure_json_file(path: Path) -> None:
    DATA_DIR.mkdir(parents=True, exist_ok=True)

    if not path.exists():
        path.write_text("[]", encoding="utf-8")


def _load_json_list(path: Path) -> list[dict[str, Any]]:
    _ensure_json_file(path)

    try:
        content = path.read_text(encoding="utf-8").strip()
        if not content:
            return []

        data = json.loads(content)

        if not isinstance(data, list):
            raise AppError(
                code="DOCUMENT_STORE_ERROR",
                message="JSON 存储格式错误",
                detail=f"{path} 内容不是数组",
                status_code=500,
            )

        return data

    except json.JSONDecodeError as exc:
        raise AppError(
            code="DOCUMENT_STORE_ERROR",
            message="读取 JSON 失败",
            detail=f"{path}: {exc}",
            status_code=500,
        ) from exc


def _save_json_list(path: Path, items: list[dict[str, Any]]) -> None:
    _ensure_json_file(path)

    try:
        path.write_text(
            json.dumps(items, ensure_ascii=False, indent=2),
            encoding="utf-8",
        )
    except OSError as exc:
        raise AppError(
            code="DOCUMENT_STORE_ERROR",
            message="写入 JSON 失败",
            detail=f"{path}: {exc}",
            status_code=500,
        ) from exc


def load_documents() -> list[dict[str, Any]]:
    return _load_json_list(DOCUMENTS_FILE)


def save_documents(documents: list[dict[str, Any]]) -> None:
    _save_json_list(DOCUMENTS_FILE, documents)


def create_document(
    filename: str,
    file_type: str,
    source_path: str,
    content: str,
    status: str = "pending",
    error_message: str | None = None,
) -> dict[str, Any]:
    now = _now_iso()

    document = {
        "id": str(uuid.uuid4()),
        "filename": filename,
        "file_type": file_type,
        "source_path": source_path,
        "content": content,
        "status": status,
        "chunk_count": 0,
        "char_count": len(content),
        "error_message": error_message,
        "created_at": now,
        "updated_at": now,
    }

    documents = load_documents()
    documents.append(document)
    save_documents(documents)

    return document


def update_document(
    document_id: str,
    updates: dict[str, Any],
) -> dict[str, Any]:
    documents = load_documents()

    for document in documents:
        if document["id"] == document_id:
            document.update(updates)
            document["updated_at"] = _now_iso()
            save_documents(documents)
            return document

    raise AppError(
        code="DOCUMENT_NOT_FOUND",
        message="文档不存在",
        detail=f"document_id={document_id}",
        status_code=404,
    )


def get_document(document_id: str) -> dict[str, Any] | None:
    documents = load_documents()

    for document in documents:
        if document["id"] == document_id:
            return document

    return None


def list_documents() -> list[dict[str, Any]]:
    documents = load_documents()
    return sorted(
        documents,
        key=lambda item: item.get("updated_at", ""),
        reverse=True,
    )


def delete_document(document_id: str) -> None:
    documents = load_documents()
    next_documents = [
        document for document in documents if document["id"] != document_id
    ]

    if len(next_documents) == len(documents):
        raise AppError(
            code="DOCUMENT_NOT_FOUND",
            message="文档不存在",
            detail=f"document_id={document_id}",
            status_code=404,
        )

    save_documents(next_documents)


def load_document_chunks() -> list[dict[str, Any]]:
    return _load_json_list(DOCUMENT_CHUNKS_FILE)


def save_document_chunks(chunks: list[dict[str, Any]]) -> None:
    _save_json_list(DOCUMENT_CHUNKS_FILE, chunks)


def create_document_chunks(
    document_id: str,
    chunks: list[dict[str, Any]],
) -> list[dict[str, Any]]:
    now = _now_iso()

    records: list[dict[str, Any]] = []

    for index, chunk in enumerate(chunks):
        content = chunk.get("content", "").strip()
        if not content:
            continue

        record = {
            "id": str(uuid.uuid4()),
            "document_id": document_id,
            "chunk_index": index,
            "heading": chunk.get("heading"),
            "content": content,
            "char_count": len(content),
            "estimated_tokens": int(chunk.get("estimated_tokens") or 0),
            "metadata": chunk.get("metadata") or {},
            "created_at": now,
        }
        records.append(record)

    existing_chunks = load_document_chunks()
    existing_chunks.extend(records)
    save_document_chunks(existing_chunks)

    return records


def list_document_chunks(document_id: str) -> list[dict[str, Any]]:
    chunks = load_document_chunks()
    document_chunks = [
        chunk for chunk in chunks if chunk["document_id"] == document_id
    ]

    return sorted(
        document_chunks,
        key=lambda item: item.get("chunk_index", 0),
    )


def delete_document_chunks(document_id: str) -> None:
    chunks = load_document_chunks()
    next_chunks = [
        chunk for chunk in chunks if chunk["document_id"] != document_id
    ]
    save_document_chunks(next_chunks)
```

## 参考实现：`document_loader.py`

```python
import re
import uuid
from pathlib import Path

from src.app.exceptions import AppError
from src.app.paths import UPLOAD_DIR

ALLOWED_DOCUMENT_EXTENSIONS = {".md", ".txt"}
MAX_DOCUMENT_FILE_SIZE_BYTES = 5 * 1024 * 1024


class DocumentLoader:
    def validate_file(self, filename: str, content: bytes) -> str:
        if not filename:
            raise AppError(
                code="DOCUMENT_INVALID_FILE",
                message="文件名不能为空",
                status_code=400,
            )

        suffix = Path(filename).suffix.lower()

        if suffix not in ALLOWED_DOCUMENT_EXTENSIONS:
            raise AppError(
                code="DOCUMENT_UNSUPPORTED_FILE_TYPE",
                message="暂只支持 .md 和 .txt 文件",
                detail=f"filename={filename}",
                status_code=400,
            )

        if not content:
            raise AppError(
                code="DOCUMENT_EMPTY_FILE",
                message="上传文件不能为空",
                status_code=400,
            )

        if len(content) > MAX_DOCUMENT_FILE_SIZE_BYTES:
            raise AppError(
                code="DOCUMENT_FILE_TOO_LARGE",
                message="上传文件过大",
                detail=f"max_size_bytes={MAX_DOCUMENT_FILE_SIZE_BYTES}",
                status_code=400,
            )

        return suffix.lstrip(".")

    def save_upload_file(self, filename: str, content: bytes) -> Path:
        UPLOAD_DIR.mkdir(parents=True, exist_ok=True)

        safe_name = self._safe_filename(filename)
        saved_name = f"{uuid.uuid4()}_{safe_name}"
        saved_path = UPLOAD_DIR / saved_name

        saved_path.write_bytes(content)

        return saved_path

    def load_text(self, filename: str, content: bytes) -> str:
        self.validate_file(filename=filename, content=content)

        try:
            text = content.decode("utf-8")
        except UnicodeDecodeError as exc:
            raise AppError(
                code="DOCUMENT_DECODE_ERROR",
                message="文件文本解码失败，请确认文件是 UTF-8 编码",
                detail=str(exc),
                status_code=400,
            ) from exc

        text = text.strip()

        if not text:
            raise AppError(
                code="DOCUMENT_EMPTY_TEXT",
                message="文件解析后文本为空",
                status_code=400,
            )

        return text

    def _safe_filename(self, filename: str) -> str:
        name = Path(filename).name
        return re.sub(r"[^a-zA-Z0-9._\-\u4e00-\u9fff]", "_", name)
```

```python
import re
from typing import Any

CHUNK_MAX_CHARS = 1000
CHUNK_OVERLAP_CHARS = 120
MIN_CHUNK_CHARS = 80


class ChunkSplitter:
    def split(self, text: str, file_type: str) -> list[dict[str, Any]]:
        normalized_text = text.strip()

        if not normalized_text:
            return []

        if file_type == "md":
            return self.split_markdown(normalized_text)

        if file_type == "txt":
            return self.split_plain_text(normalized_text)

        return self.split_plain_text(normalized_text)

    def split_markdown(self, text: str) -> list[dict[str, Any]]:
        heading_pattern = re.compile(r"^(#{1,6})\s+(.+)$", re.MULTILINE)

        matches = list(heading_pattern.finditer(text))

        if not matches:
            return self.split_plain_text(text)

        chunks: list[dict[str, Any]] = []

        for index, match in enumerate(matches):
            heading = match.group(2).strip()
            heading_level = len(match.group(1))

            section_start = match.end()
            section_end = matches[index + 1].start() if index + 1 < len(matches) else len(text)

            section_content = text[section_start:section_end].strip()

            if not section_content:
                continue

            section_chunks = self._split_by_paragraphs(
                text=section_content,
                heading=heading,
                metadata={
                    "split_method": "markdown_heading",
                    "heading_level": heading_level,
                },
            )

            chunks.extend(section_chunks)

        return chunks

    def split_plain_text(self, text: str) -> list[dict[str, Any]]:
        return self._split_by_paragraphs(
            text=text,
            heading=None,
            metadata={
                "split_method": "paragraph",
            },
        )

    def _split_by_paragraphs(
        self,
        text: str,
        heading: str | None,
        metadata: dict[str, Any],
    ) -> list[dict[str, Any]]:
        paragraphs = [
            paragraph.strip()
            for paragraph in re.split(r"\n\s*\n", text)
            if paragraph.strip()
        ]

        chunks: list[dict[str, Any]] = []
        buffer: list[str] = []
        buffer_chars = 0

        for paragraph in paragraphs:
            paragraph_len = len(paragraph)

            if paragraph_len > CHUNK_MAX_CHARS:
                if buffer:
                    chunks.append(
                        self._build_chunk(
                            content="\n\n".join(buffer),
                            heading=heading,
                            metadata=metadata,
                        )
                    )
                    buffer = []
                    buffer_chars = 0

                chunks.extend(
                    self.split_long_text(
                        text=paragraph,
                        heading=heading,
                        metadata={
                            **metadata,
                            "split_method": "length_fallback",
                        },
                    )
                )
                continue

            next_chars = buffer_chars + paragraph_len

            if buffer and next_chars > CHUNK_MAX_CHARS:
                chunks.append(
                    self._build_chunk(
                        content="\n\n".join(buffer),
                        heading=heading,
                        metadata=metadata,
                    )
                )
                buffer = [paragraph]
                buffer_chars = paragraph_len
            else:
                buffer.append(paragraph)
                buffer_chars = next_chars

        if buffer:
            chunks.append(
                self._build_chunk(
                    content="\n\n".join(buffer),
                    heading=heading,
                    metadata=metadata,
                )
            )

        return [
            chunk
            for chunk in chunks
            if len(chunk["content"]) >= MIN_CHUNK_CHARS or len(chunks) == 1
        ]

    def split_long_text(
        self,
        text: str,
        heading: str | None = None,
        metadata: dict[str, Any] | None = None,
    ) -> list[dict[str, Any]]:
        chunks: list[dict[str, Any]] = []

        start = 0
        text_length = len(text)
        step = max(1, CHUNK_MAX_CHARS - CHUNK_OVERLAP_CHARS)

        while start < text_length:
            end = min(start + CHUNK_MAX_CHARS, text_length)
            content = text[start:end].strip()

            if content:
                chunks.append(
                    self._build_chunk(
                        content=content,
                        heading=heading,
                        metadata=metadata or {"split_method": "length_fallback"},
                    )
                )

            if end >= text_length:
                break

            start += step

        return chunks

    def estimate_tokens(self, text: str) -> int:
        if not text:
            return 0

        chinese_chars = sum(1 for ch in text if "\u4e00" <= ch <= "\u9fff")
        other_chars = len(text) - chinese_chars

        return chinese_chars + max(1, other_chars // 4)

    def _build_chunk(
        self,
        content: str,
        heading: str | None,
        metadata: dict[str, Any],
    ) -> dict[str, Any]:
        normalized_content = content.strip()

        return {
            "heading": heading,
            "content": normalized_content,
            "char_count": len(normalized_content),
            "estimated_tokens": self.estimate_tokens(normalized_content),
            "metadata": metadata,
        }
```

## 逐行解释

### `split`

统一入口。

根据 `file_type` 分发：

```text
md -> split_markdown
txt -> split_plain_text
```

### `heading_pattern`

```python
r"^(#{1,6})\s+(.+)$"
```

匹配 Markdown 标题：

```text
# 一级标题
## 二级标题
### 三级标题
```

### `match.group(2)`

取标题文字。

例如：

```markdown
## JSONResponse
```

得到：

```text
JSONResponse
```

### `section_start` / `section_end`

用于截取当前标题到下一个标题之间的内容。

### `_split_by_paragraphs`

段落比固定长度更自然。

段落通常代表一个较完整语义单元。

### `split_long_text`

兜底切分。

当单个段落超过 `CHUNK_MAX_CHARS`，必须强制拆开，否则后续 embedding 太长。

### `step = CHUNK_MAX_CHARS - CHUNK_OVERLAP_CHARS`

例如：

```text
max = 1000
overlap = 120
step = 880
```

第二个 chunk 会从第 880 个字符开始，和上一个 chunk 重叠 120 字符。

### `MIN_CHUNK_CHARS`

过滤极短 chunk。

但如果全文只有一个 chunk，即使小于 80 字，也保留。

## 目标

## 核心思想

`DocumentService` 是编排层。

它不亲自解析文件，不亲自切分文本，不亲自操作 JSON 细节，而是组织它们。

---

## 为什么

如果你把所有逻辑都写在一个函数里，会变成：

```text

校验大小
保存文件
decode utf-8
创建 document
正则切 markdown

写 chunks json
处理错误
返回响应
```

这就是巨型函数。

真实工程要拆成：

```text
DocumentLoader：外部文件 -> text
ChunkSplitter：text -> chunks
document_store：dict -> JSON
DocumentService：编排业务流程
routes.py：HTTP 参数接收和响应包装
```

类比 NestJS：

```text
Controller -> Service -> Repository
```

## 参考实现：`document_service.py`

```python
from pathlib import Path
from typing import Any

from fastapi import UploadFile

from src.app.document_store import (
    create_document,
    create_document_chunks,
    delete_document,
    delete_document_chunks,
    get_document,
    list_document_chunks,
    list_documents,
    update_document,
)
from src.app.exceptions import AppError
from src.app.services.chunk_splitter import ChunkSplitter
from src.app.services.document_loader import DocumentLoader


class DocumentService:
    def __init__(self) -> None:
        self.loader = DocumentLoader()
        self.splitter = ChunkSplitter()

    async def upload_document(self, file: UploadFile) -> dict[str, Any]:
        filename = file.filename or ""

        document: dict[str, Any] | None = None

        try:
            content_bytes = await file.read()

            file_type = self.loader.validate_file(
                filename=filename,
                content=content_bytes,
            )

            text = self.loader.load_text(
                filename=filename,
                content=content_bytes,
            )

            saved_path = self.loader.save_upload_file(
                filename=filename,
                content=content_bytes,
            )

            document = create_document(
                filename=filename,
                file_type=file_type,
                source_path=str(saved_path),
                content=text,
                status="processing",
            )

            chunk_drafts = self.splitter.split(
                text=text,
                file_type=file_type,
            )

            if not chunk_drafts:
                raise AppError(
                    code="DOCUMENT_CHUNK_EMPTY",
                    message="文档没有生成有效 Chunk",
                    status_code=400,
                )

            chunk_records = create_document_chunks(
                document_id=document["id"],
                chunks=chunk_drafts,
            )

            updated_document = update_document(
                document["id"],
                {
                    "status": "completed",
                    "chunk_count": len(chunk_records),
                    "char_count": len(text),
                    "error_message": None,
                },
            )

            return {
                "document_id": updated_document["id"],
                "filename": updated_document["filename"],
                "file_type": updated_document["file_type"],
                "status": updated_document["status"],
                "chunk_count": updated_document["chunk_count"],
                "char_count": updated_document["char_count"],
                "created_at": updated_document["created_at"],
            }

        except Exception as exc:
            if document is not None:
                update_document(
                    document["id"],
                    {
                        "status": "failed",
                        "error_message": str(exc),
                    },
                )

            raise

    def list_documents(self) -> list[dict[str, Any]]:
        documents = list_documents()

        return [
            self._without_content(document)
            for document in documents
        ]

    def get_document_detail(self, document_id: str) -> dict[str, Any]:
        document = get_document(document_id)

        if document is None:
            raise AppError(
                code="DOCUMENT_NOT_FOUND",
                message="文档不存在",
                detail=f"document_id={document_id}",
                status_code=404,
            )

        return document

    def get_document_chunks(self, document_id: str) -> list[dict[str, Any]]:
        document = get_document(document_id)

        if document is None:
            raise AppError(
                code="DOCUMENT_NOT_FOUND",
                message="文档不存在",
                detail=f"document_id={document_id}",
                status_code=404,
            )

        return list_document_chunks(document_id)

    def delete_document(self, document_id: str) -> dict[str, Any]:
        document = get_document(document_id)

        if document is None:
            raise AppError(
                code="DOCUMENT_NOT_FOUND",
                message="文档不存在",
                detail=f"document_id={document_id}",
                status_code=404,
            )

        source_path = document.get("source_path")
        if source_path:
            path = Path(source_path)
            if path.exists() and path.is_file():
                path.unlink()

        delete_document_chunks(document_id)
        delete_document(document_id)

        return {
            "success": True,
            "message": "文档已删除",
        }

    def _without_content(self, document: dict[str, Any]) -> dict[str, Any]:
        return {
            key: value
            for key, value in document.items()
            if key != "content"
        }
```

---

## 逐行解释

### `filename = file.filename or ""`

`UploadFile.filename` 可能为空。

所以先兜底为空字符串，然后交给 loader 校验。

### `content_bytes = await file.read()`

上传文件读取是异步操作。

因为 FastAPI 上传文件来自 HTTP 请求流，使用 `await` 不会阻塞事件循环。

### 为什么先 `create_document` 再切 chunk

流程是：

```text
先创建 document
再切 chunks
```

而不是：

```text
先切 chunks
再创建 document
```

原因：

```text
真实系统中 document 是任务主记录
即使后续解析失败，也应该知道这次上传发生过
status/error_message 才有地方记录
```

第五周当前 loader 校验失败前还没创建 document，这是可以接受的。

后续如果要记录所有失败上传，可以更早创建 `pending`。

### 为什么 status 是状态机

```text
processing -> completed
processing -> failed
```

真实系统里这里会变成异步任务：

```text
上传接口立即返回 pending
后台 worker 解析
完成后更新 completed
失败后更新 failed
```

第五周先同步处理，降低复杂度。

### `_without_content`

列表接口不返回 content。

防止文档多时响应过大。

## curl 解释

### `-X`

指定 HTTP 方法。

例如：

```bash
-X POST
-X DELETE
```

### `-F`

发送表单字段。

上传文件时：

```bash
-F "file=@/tmp/fastapi-guide.md"
```

表示：

```text
字段名叫 file
字段内容来自 /tmp/fastapi-guide.md 这个文件
```

### `multipart/form-data`

浏览器上传文件时常用的请求格式。

它可以同时传：

```text
文本字段
文件字段
二进制内容
```

### 为什么不用 JSON

不要这样：

```bash
-H "Content-Type: application/json"
```

因为文件上传不是 JSON body。
