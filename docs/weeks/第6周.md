# 4. 为什么 Embedding 是 RAG 的前置工程

## 4.1 Chat Model 和 Embedding Model 的区别

| 类型              | 输入                | 输出     | 用途   |
| --------------- | ----------------- | ------ | ---- |
| Chat Model      | messages / prompt | 自然语言回答 | 生成答案 |
| Embedding Model | text / texts      | 数字向量   | 语义检索 |

你的 `OllamaProvider` 是 Chat Provider：

```text
messages -> answer
messages -> stream delta
```

Embedding Provider 是：

```text
text -> list[float]
texts -> list[list[float]]
```

两者不是一回事。

---

## 4.2 为什么 chunk 要转成向量

用户问：

```text
FastAPI 如何统一处理错误？
```

文档写：

```text
FastAPI 可以通过 exception_handler 注册全局异常处理器。
```

关键词不完全一样，但语义接近。

Embedding 的作用是：

```text
把文本变成向量，让语义接近的文本在向量空间中距离更近。
```

---

## 4.3 向量维度是什么

一个向量：

```python
[0.12, -0.05, 0.87]
```

这是 3 维。

本周推荐模型：

```text
sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
```

通常输出 384 维向量。

所以 PostgreSQL 字段使用：

```sql
embedding vector(384)
```

`vector(384)` 表示：

```text
这个字段只能保存 384 个 float 组成的向量。
```

如果换成 768 维模型，数据库字段也要变成：

```sql
embedding vector(768)
```

否则写入失败。

---

## 4.4 为什么模型换了，向量库通常要重建

不同 embedding 模型生成的向量空间不同。

即使都是 384 维，也不代表能混用。

例如：

```text
模型 A 生成的向量空间
模型 B 生成的向量空间
```

它们的坐标含义可能不同。

所以必须存：

```text
embedding_model
```

并在模型切换后重新生成 embedding。

---

## 4.5 为什么需要 embedding_status

向量化不是瞬间完成的，可能失败：

```text
模型下载失败
torch 加载失败
维度不一致
数据库写入失败
```

所以每个 chunk 需要状态：

```text
pending      等待向量化
processing   正在处理
completed    已完成
failed       失败
```

同时保存：

```text
embedding_error
embedding_updated_at
```

# 5. 核心概念

## 5.1 Embedding

Embedding 是文本的数字表达。

```text
文本：
FastAPI 可以通过 exception_handler 注册全局异常处理器。

向量：
[0.0123, -0.0456, 0.0789, ...]
```

用途：

```text
语义搜索
推荐
聚类
去重
RAG 检索
```

---

## 5.2 Vector

Vector 就是向量。

Python 中：

```python
list[float]
```

多个向量：

```python
list[list[float]]
```

类比 TypeScript：

```ts
number[]
number[][]
```

---

## 5.3 pgvector

pgvector 是 PostgreSQL 的向量扩展。

它让 PostgreSQL 支持：

```text
保存 vector
计算向量距离
按相似度排序
建立向量索引
```

常用运算符：

```text
<->   L2 distance
<#>   inner product distance
<=>   cosine distance
```

本周主要用：

```sql
embedding <=> query_embedding
```

表示 cosine distance。

结论：

```text
distance 越小，越相似。
score = 1 - distance。
score 越大，越相似。
```

---

## 5.4 VectorStore

VectorStore 是向量存储层。

本项目中：

```text
PgVectorStore
```

负责：

```text
同步 JSON chunks
查询 chunks
更新 embedding 状态
写入 embedding
执行 search debug
```

它不负责：

```text
加载模型
调用 LLM
处理 HTTP 请求
```

---

## 5.5 EmbeddingProvider

EmbeddingProvider 是 embedding 模型抽象。

```python
class EmbeddingProvider:
    def embed_text(self, text: str) -> list[float]:
        ...

    def embed_texts(self, texts: list[str]) -> list[list[float]]:
        ...

    def dimension(self) -> int:
        ...
```

类比前端：

```ts
interface EmbeddingProvider {
  embedText(text: string): number[]
  embedTexts(texts: string[]): number[][]
  dimension(): number
}
```

## 代码骨架：`sql/init.sql`

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE IF NOT EXISTS document_chunks (
    id TEXT PRIMARY KEY,
    document_id TEXT NOT NULL,
    chunk_index INTEGER NOT NULL,
    heading TEXT,
    content TEXT NOT NULL,
    char_count INTEGER NOT NULL,
    estimated_tokens INTEGER NOT NULL,

    embedding_model TEXT,

    embedding_error TEXT,
    embedding_updated_at TIMESTAMP,

CREATE INDEX IF NOT EXISTS idx_document_chunks_document_id
```

## 逐行解释 SQL

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

启用 pgvector。

```sql
CREATE TABLE IF NOT EXISTS document_chunks
```

如果表不存在就创建。

```sql
id TEXT PRIMARY KEY
```

直接复用 JSON chunk 里的 UUID 字符串。

```sql
document_id TEXT NOT NULL
```

保留 chunk 属于哪个 document。

```sql
chunk_index INTEGER NOT NULL
```

保留 chunk 在文档中的顺序。

```sql
metadata JSONB NOT NULL DEFAULT '{}'::jsonb
```

保存 chunk 的额外信息，例如：

```json
{
  "split_method": "markdown_heading",
  "heading_level": 2
}
```

```sql
embedding vector(384)
```

保存 384 维向量。

```sql
embedding_model TEXT
```

记录是哪个 embedding 模型生成的。

```sql
embedding_status TEXT NOT NULL DEFAULT 'pending'
```

记录向量化状态。

```sql
embedding_error TEXT
```

记录失败原因。

```sql
embedding_updated_at TIMESTAMP
```

记录 embedding 更新时间。

```sql
created_at TIMESTAMP NOT NULL DEFAULT NOW()
updated_at TIMESTAMP NOT NULL DEFAULT NOW()
```

`created_at` 是行创建时间。

`updated_at` 是行最后更新时间。

```python
import json
from typing import Any

from pgvector.psycopg import register_vector

from src.app.db import get_connection


class PgVectorStore:
    def _register_vector(self, conn) -> None:
        register_vector(conn)

    def sync_chunks_from_json(self, chunks: list[dict[str, Any]]) -> int:
        synced_count = 0

        with get_connection() as conn:
            self._register_vector(conn)

            with conn.cursor() as cur:
                for chunk in chunks:
                    cur.execute(
                        """
                        INSERT INTO document_chunks (
                            id,
                            document_id,
                            chunk_index,
                            heading,
                            content,
                            char_count,
                            estimated_tokens,
                            metadata,
                            updated_at
                        )
                        VALUES (
                            %(id)s,
                            %(document_id)s,
                            %(chunk_index)s,
                            %(heading)s,
                            %(content)s,
                            %(char_count)s,
                            %(estimated_tokens)s,
                            %(metadata)s::jsonb,
                            NOW()
                        )
                        ON CONFLICT (id) DO UPDATE SET
                            document_id = EXCLUDED.document_id,
                            chunk_index = EXCLUDED.chunk_index,
                            heading = EXCLUDED.heading,
                            content = EXCLUDED.content,
                            char_count = EXCLUDED.char_count,
                            estimated_tokens = EXCLUDED.estimated_tokens,
                            metadata = EXCLUDED.metadata,
                            updated_at = NOW()
                        """,
                        {
                            "id": chunk["id"],
                            "document_id": chunk["document_id"],
                            "chunk_index": chunk["chunk_index"],
                            "heading": chunk.get("heading"),
                            "content": chunk["content"],
                            "char_count": chunk["char_count"],
                            "estimated_tokens": chunk["estimated_tokens"],
                            "metadata": json.dumps(
                                chunk.get("metadata") or {},
                                ensure_ascii=False,
                            ),
                        },
                    )
                    synced_count += 1

            conn.commit()

        return synced_count
```

```python
from sentence_transformers import SentenceTransformer

from src.app.config import get_embedding_model
from src.app.services.embedding.base import EmbeddingProvider


class SentenceTransformerEmbeddingProvider(EmbeddingProvider):
    def __init__(self, model_name: str | None = None) -> None:
        self.model_name = model_name or get_embedding_model()
        self.model = SentenceTransformer(self.model_name)

    def embed_text(self, text: str) -> list[float]:
        embeddings = self.embed_texts([text])
        return embeddings[0]

    def embed_texts(self, texts: list[str]) -> list[list[float]]:
        vectors = self.model.encode(
            texts,
            normalize_embeddings=True,
        )
        return vectors.tolist()

    def dimension(self) -> int:
        dimension = self.model.get_sentence_embedding_dimension()
        if dimension is None:
            raise RuntimeError("无法获取 embedding 模型维度")
        return int(dimension)
```

```python
                    SELECT *
                    FROM document_chunks

                    """,

                        "embedding": embedding,
                        "embedding_model": embedding_model,

                    UPDATE document_chunks
                    SET embedding_status = 'failed',
                        embedding_error = %(error_message)s,
                        embedding_updated_at = NOW(),

                        "error_message": error_message,

        with get_connection() as conn:
            with conn.cursor() as cur:
                cur.execute(
                    """
                    SELECT
                        id AS chunk_id,
                        chunk_index,
                        embedding_status,
                        embedding_model,
                        embedding_error,
                        embedding_updated_at
                    FROM document_chunks
                    WHERE document_id = %(document_id)s
                    ORDER BY chunk_index ASC
                    """,
                    {"document_id": document_id},
                )
                return list(cur.fetchall())

    def search_similar_chunks(
        self,
        query_embedding: list[float],
        top_k: int = 5,
    ) -> list[dict[str, Any]]:
        with get_connection() as conn:
            self._register_vector(conn)

            with conn.cursor() as cur:
                cur.execute(
                    """
                    SELECT
                        id AS chunk_id,
                        document_id,
                        chunk_index,
                        heading,
                        content,
                        embedding <=> %(query_embedding)s AS distance,
                        1 - (embedding <=> %(query_embedding)s) AS score
                    FROM document_chunks
                    WHERE embedding IS NOT NULL
                      AND embedding_status = 'completed'
                    ORDER BY embedding <=> %(query_embedding)s
                    LIMIT %(top_k)s
                    """,
                    {
                        "query_embedding": query_embedding,
                        "top_k": top_k,
                    },
                )
```

```python
        return {
            "text": text,
            "embedding_model": get_embedding_model(),
            "dimension": len(vector),
            "vector_preview": [round(value, 4) for value in vector[:5]],
            "vector_norm": round(vector_norm, 4),
        }

    def embed_document(self, document_id: str) -> dict[str, Any]:
        all_chunks = load_document_chunks()
        self.vector_store.sync_chunks_from_json(all_chunks)

        chunks = self.vector_store.list_chunks_by_document(document_id)

        if not chunks:
            raise AppError(
                code="DOCUMENT_CHUNKS_NOT_FOUND",
                message="该文档没有可向量化的 chunks",
                detail=f"document_id={document_id}",
                status_code=404,
            )

        return self._embed_chunks(
            chunks=chunks,
            document_id=document_id,
        )

    def embed_all_pending(self) -> dict[str, Any]:
        all_chunks = load_document_chunks()
        self.vector_store.sync_chunks_from_json(all_chunks)

        limit = get_embedding_batch_size()
        chunks = self.vector_store.list_pending_chunks(limit=limit)

        result = self._embed_chunks(chunks=chunks)

        return {
            "total_chunks": result["total_chunks"],
            "embedded_chunks": result["embedded_chunks"],
            "failed_chunks": result["failed_chunks"],
            "embedding_model": result["embedding_model"],
            "status": result["status"],
        }

    def get_document_embedding_status(self, document_id: str) -> dict[str, Any]:
        items = self.vector_store.get_document_embedding_status(document_id)

        return {
            "document_id": document_id,
            "items": [
                {
                    **item,
                    "embedding_updated_at": (
                        item["embedding_updated_at"].isoformat()
                        if item.get("embedding_updated_at")
                        else None
                    ),
                }
                for item in items
            ],
        }

    def search_debug(self, query: str, top_k: int = 5) -> dict[str, Any]:
        query_embedding = self.provider.embed_text(query)
        self._validate_dimension(query_embedding)

        items = self.vector_store.search_similar_chunks(
            query_embedding=query_embedding,
            top_k=top_k,
        )

        return {
            "query": query,
            "embedding_model": get_embedding_model(),
            "top_k": top_k,
            "items": [
                {
                    **item,
                    "distance": round(float(item["distance"]), 4),
                    "score": round(float(item["score"]), 4),
                }
                for item in items
            ],
        }

    def _embed_chunks(
        self,
        chunks: list[dict[str, Any]],
        document_id: str | None = None,
    ) -> dict[str, Any]:
        total_chunks = len(chunks)
        embedded_chunks = 0
        failed_chunks = 0

        if not chunks:
            return {
                "document_id": document_id,
                "total_chunks": 0,
                "embedded_chunks": 0,
                "failed_chunks": 0,
                "embedding_model": get_embedding_model(),
                "status": "completed",
            }

        chunk_ids = [chunk["id"] for chunk in chunks]
        texts = [chunk["content"] for chunk in chunks]

        self.vector_store.mark_chunks_processing(chunk_ids)

        try:
            embeddings = self.provider.embed_texts(texts)

            for chunk, embedding in zip(chunks, embeddings):
                self._validate_dimension(embedding)

                self.vector_store.update_chunk_embedding(
                    chunk_id=chunk["id"],
                    embedding=embedding,
                    embedding_model=get_embedding_model(),
                )
                embedded_chunks += 1

        except Exception as exc:
            failed_chunks = total_chunks - embedded_chunks

            for chunk in chunks[embedded_chunks:]:
                self.vector_store.mark_chunk_failed(
                    chunk_id=chunk["id"],
                    error_message=str(exc),
                )

        status = "completed" if failed_chunks == 0 else "failed"

        return {
            "document_id": document_id,
            "total_chunks": total_chunks,
            "embedded_chunks": embedded_chunks,
            "failed_chunks": failed_chunks,
            "embedding_model": get_embedding_model(),
            "status": status,
        }

    def _validate_dimension(self, vector: list[float]) -> None:
        expected_dimension = get_embedding_dimension()
        actual_dimension = len(vector)

        if actual_dimension != expected_dimension:
            raise AppError(
                code="EMBEDDING_DIMENSION_MISMATCH",
                message="Embedding 维度和配置不一致",
                detail=f"expected={expected_dimension}, actual={actual_dimension}",
                status_code=500,
            )


@lru_cache
def get_embedding_service() -> EmbeddingService:
    return EmbeddingService()
```

# Day 6：实现 embed-all 和 search-debug

## 目标

```text
实现 /documents/embed-all
实现 /embeddings/search-debug
用 query embedding 检索相似 chunks
```

---

## 先讲为什么

`embed_document` 适合处理一个文档。

`embed-all` 适合处理所有待向量化 chunks：

```text
pending chunks -> batch embedding -> completed
```

`search-debug` 用于验证检索质量：

```text
query -> query embedding -> pgvector similarity search -> top_k chunks
```

这里要注意：查询文本 `query` 不是直接与数据库文本比对，必须先通过 EmbeddingProvider 转成向量 `query_embedding`，再与 `document_chunks.embedding` 做相似度计算。

它不是 RAG，因为它不调用 LLM。

---

## SQL 解释

核心 SQL：

```sql
SELECT
    id AS chunk_id,
    document_id,
    chunk_index,
    heading,
    content,
    embedding <=> %(query_embedding)s AS distance,
    1 - (embedding <=> %(query_embedding)s) AS score
FROM document_chunks
WHERE embedding IS NOT NULL
  AND embedding_status = 'completed'
ORDER BY embedding <=> %(query_embedding)s
LIMIT %(top_k)s
```

解释：

`query_embedding` 是从查询文本 `query` 生成的向量，不是原始字符串直接与数据库比对。

```sql
embedding <=> query_embedding
```

计算 cosine distance。

```sql
ORDER BY embedding <=> query_embedding
```

距离越小越靠前。

```sql
LIMIT top_k
```

只返回前 k 条。

```sql
1 - distance AS score
```

把 distance 转成更直观的相似分数。

# 15. 常见报错排查

| 问题                                        | 原因                    | 处理                                                 |
| ----------------------------------------- | --------------------- | -------------------------------------------------- |
| `psql: command not found`                 | psql 没进 PATH          | 配置 PostgreSQL bin 到 PATH                           |
| `connection refused`                      | PostgreSQL 没启动        | `brew services start postgresql@16`                |
| `role does not exist`                     | 用户没创建                 | `CREATE USER wangzilong ...`                  |
| `database does not exist`                 | 数据库没创建                | `CREATE DATABASE ai_backend OWNER wangzilong` |
| `password authentication failed`          | 密码错                   | `ALTER USER wangzilong WITH PASSWORD ...`     |
| `extension "vector" is not available`     | pgvector 未安装          | `brew install pgvector`                            |
| `permission denied to create extension`   | 当前用户权限不足              | 用管理用户创建 extension                                  |
| `permission denied for schema public`     | schema 权限不足           | `GRANT USAGE, CREATE ON SCHEMA public ...`         |
| `relation document_chunks does not exist` | 没执行 init.sql          | `psql ... -f sql/init.sql`                         |
| sentence-transformers 安装慢                 | torch 等依赖大            | 正常等待                                               |
| torch 安装失败                                | Python 版本不兼容          | 使用 Python 3.11 / 3.12                              |
| 模型下载失败                                    | Hugging Face 网络问题     | 配代理或镜像                                             |
| 第一次 embedding 很慢                          | 模型首次加载                | 正常                                                 |
| 维度不一致                                     | 模型不是 384 维            | 修改 `EMBEDDING_DIMENSION` 和 `vector(n)`             |
| vector 写入失败                               | 没注册 pgvector 类型       | `register_vector(conn)`                            |
| search-debug 空结果                          | 没 completed embedding | 查 `embedding_status`                               |
| 中文检索差                                     | 模型或 chunk 问题          | 先看召回内容，再考虑换模型                                      |

# 17. 本周必须掌握的 AI 工程概念

```text
1. Embedding 是文本的向量表示。

9. embedding_status 用于任务状态管理。
10. batch embedding 比逐条 embedding 高效。
11. search-debug 是检索调试，不是 RAG。
```

