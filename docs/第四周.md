## 3. 为什么 AI Chat 必须管理上下文窗口

### 为什么不能把所有 Message 无限塞给模型

大模型每次调用本质上都是：

```text
你把一组 messages 发给模型
模型根据这些 messages 生成下一段回答
```

如果历史会话越来越长，直接全部传给模型会有几个问题：

| 问题      | 影响               |
| ------- | ---------------- |
| 上下文长度有限 | 超过模型窗口会报错或被截断    |
| 成本增加    | token 越多，调用越慢、越贵 |
| 噪音增加    | 很早以前无关内容会干扰当前回答  |
| 延迟增加    | prompt 越长，模型处理越慢 |
| 调试困难    | 你不知道模型到底看到了什么    |

所以真实 AI Chat 后端一定要做：

```text
Context Window Management
```

即：

```text
控制每次真正传给模型的上下文内容。
```

## 4. Context Window 和 Summary Memory 的核心概念

### 4.1 Context Window 是什么

Context Window 不是“最近 10 条消息”的同义词。

更准确地说：

```text
Context Window = 本次请求真正传给模型的上下文集合。
```

在第四周第一版中：

```text
LLM messages =
  1. system prompt
  2. conversation.summary，如果存在
  3. 最近 N 条原始 messages
```

注意：你的 Week 3 流程是：

```text
先保存 user message
再 build_llm_messages
再调用模型
```

所以 `ContextBuilder` 读取最近消息时，已经包含当前用户消息。

因此：

```text
不要再 append 当前 user message。
```

否则会出现：

```text
用户问题重复传两次
```

### 4.2 Summary Memory 是什么

Summary Memory 是一种简化的长期记忆。

它不是完整数据库记忆，也不是 RAG。

它做的事情是：

```text
把较早的对话压缩成一段自然语言摘要，
放进 conversation.summary，
后续每轮对话都把 summary 放入上下文。
```

它解决的问题是：

```text
最近消息保留细节；
更早消息保留摘要；
系统不会无限增长上下文。
```

### 4.3 system prompt / summary / recent messages 各自职责

| 部分                   | role             | 职责                        |
| -------------------- | ---------------- | ------------------------- |
| system prompt        | system           | 固定定义助手身份、回答风格、边界          |
| summary              | system           | 告诉模型“这个会话之前发生过什么”         |
| recent messages      | user / assistant | 保留最近几轮原始语义和细节             |
| current user message | user             | 当前问题，已在 recent messages 里 |

为什么 summary 也用 `system` role？

因为它不是用户新说的话，也不是助手回答，而是系统注入给模型的上下文背景。

## 逐行解释

### `ContextMessageItem`

```python
role: str
content: str
```

它只暴露最关键的两项：

```text
role：模型如何理解这条消息的身份
content：模型真正看到的文本
```

不返回 `id`、`metadata`、`created_at`，因为 context preview 的目标是：

```text
调试 prompt，而不是调试数据库。
```

### `ContextPreviewResponse.conversation_id`

用于确认你看的 preview 属于哪个会话。

### `summary`

当前会话摘要。

如果是 `None`，说明还没有 summary memory。

### `recent_message_count`

表示最近消息窗口中实际进入上下文的消息数。

不是数据库里的全部消息数。

### `estimated_tokens`

粗略估算 token。

为什么是估算？

因为不同模型 tokenizer 不一样。Ollama 后面可能跑 Qwen、Gemma、Llama，每个 tokenizer 都不同。第四周先用近似值防止上下文失控，后续可以替换为：

```text
tiktoken
transformers tokenizer
Ollama 返回的 prompt_eval_count
```

### `estimated_chars`

字符数是最稳定、最容易实现的保护指标。

第四周先用字符数限制上下文长度，比一上来做精确 token 计算更适合学习阶段。

### `max_recent_messages`

最多取最近多少条消息。

### `max_context_chars`

上下文最大字符数限制。

### `messages`

这就是最终传给 LLM Provider 的 messages。

## 参考实现

```python
# src/app/services/context_builder.py

from typing import Any

from src.app.conversation_store import get_conversation, list_recent_messages
from src.app.exceptions import ConversationError

DEFAULT_SYSTEM_PROMPT = "你是一个专业、耐心、严谨的 AI 开发学习助手。请使用简体中文回答。"

MAX_RECENT_MESSAGES = 10
MAX_CONTEXT_CHARS = 12000

ALLOWED_CONTEXT_ROLES = {"system", "user", "assistant", "tool"}


class ContextBuilder:
    def __init__(
        self,
        max_recent_messages: int = MAX_RECENT_MESSAGES,
        max_context_chars: int = MAX_CONTEXT_CHARS,
        system_prompt: str = DEFAULT_SYSTEM_PROMPT,
    ) -> None:
        self.max_recent_messages = max_recent_messages
        self.max_context_chars = max_context_chars
        self.system_prompt = system_prompt

    def _ensure_conversation_exists(self, conversation_id: str) -> dict[str, Any]:
        conversation = get_conversation(conversation_id)

        if conversation is None:
            raise ConversationError(
                message="会话不存在",
                detail=f"conversation_id={conversation_id}",
                status_code=404,
            )

        return conversation

    def estimate_tokens(self, text: str) -> int:
        if not text:
            return 0

        chinese_chars = sum(1 for ch in text if "\u4e00" <= ch <= "\u9fff")
        other_chars = len(text) - chinese_chars

        return chinese_chars + max(1, other_chars // 4)

    def estimate_messages_tokens(self, messages: list[dict[str, str]]) -> int:
        return sum(self.estimate_tokens(message["content"]) for message in messages)

    def _messages_chars(self, messages: list[dict[str, str]]) -> int:
        return sum(len(message["content"]) for message in messages)

    def build_messages(self, conversation_id: str) -> list[dict[str, str]]:
        conversation = self._ensure_conversation_exists(conversation_id)

        base_messages: list[dict[str, str]] = [
            {
                "role": "system",
                "content": self.system_prompt,
            }
        ]

        summary = conversation.get("summary")
        if summary:
            base_messages.append(
                {
                    "role": "system",
                    "content": f"当前会话摘要：{summary}",
                }
            )

        recent_messages = list_recent_messages(
            conversation_id,
            limit=self.max_recent_messages,
        )

        selected_recent_messages: list[dict[str, str]] = []
        current_chars = self._messages_chars(base_messages)

        for message in reversed(recent_messages):
            role = message.get("role")
            content = message.get("content", "")

            if role not in ALLOWED_CONTEXT_ROLES:
                continue

            normalized_message = {
                "role": role,
                "content": content,
            }

            next_chars = current_chars + len(content)

            if next_chars > self.max_context_chars:
                break

            selected_recent_messages.append(normalized_message)
            current_chars = next_chars

        selected_recent_messages.reverse()

        return base_messages + selected_recent_messages

    def build_preview(self, conversation_id: str) -> dict[str, Any]:
        conversation = self._ensure_conversation_exists(conversation_id)
        messages = self.build_messages(conversation_id)

        recent_message_count = sum(
            1
            for message in messages
            if message["role"] in {"user", "assistant", "tool"}
        )

        return {
            "conversation_id": conversation_id,
            "summary": conversation.get("summary"),
            "recent_message_count": recent_message_count,
            "estimated_tokens": self.estimate_messages_tokens(messages),
            "estimated_chars": self._messages_chars(messages),
            "max_recent_messages": self.max_recent_messages,
            "max_context_chars": self.max_context_chars,
            "messages": messages,
        }
```

## 逐行解释

### 常量

```python
DEFAULT_SYSTEM_PROMPT = "..."
```

固定 system prompt。

它应该稳定，不要每轮变化太大。system prompt 太飘，会让模型行为不稳定。

```python
MAX_RECENT_MESSAGES = 10
MAX_CONTEXT_CHARS = 12000
```

这是第四周第一版上下文策略：

```text
最多 10 条最近消息
最多 12000 字符
```

### `__init__`

```python
def __init__(..., max_recent_messages=..., max_context_chars=...):
```

这样以后你可以在测试里这样用：

```python
builder = ContextBuilder(max_recent_messages=3)
```

这和前端组件传 props 很像。

### `_ensure_conversation_exists`

和你现在 `conversation_service.py` 里的逻辑一样。

为什么这里也要校验？

因为 ContextBuilder 自己需要读取 conversation。如果会话不存在，它应该抛出业务异常，而不是返回空上下文。

### `estimate_tokens`

```python
chinese_chars = ...
other_chars = ...
```

粗略规则：

```text
中文：1 个字大致按 1 token
英文：约 4 个字符按 1 token
```

这不精确，但足够做保护。

### `build_messages`

核心流程：

```text
1. 先放 system prompt
2. 如果有 summary，再放 summary system message
3. 读取最近 N 条 messages
4. 从最新消息往前尝试加入
5. 超过字符限制就停止
6. reverse 回正常时间顺序
```

为什么从最新往前加？

因为越新的消息越重要。
如果上下文超长，应该优先保留当前用户消息和最近几轮。

### 为什么最后 reverse

因为传给模型时，消息顺序应该是：

```text
旧 -> 新
```

如果不 reverse，会变成：

```text

...
```

顺序是错的。

### `build_preview`

它复用 `build_messages`，保证 preview 和真实 LLM 调用看到的是同一套上下文。

这是关键点：

```text
不要在 preview 里重新写一套拼接逻辑。
```

否则调试接口和真实调用可能不一致。

# Day 5：实现 Summarizer 和手动摘要接口

## 目标

新增：



```text
4. 调用 OllamaProvider
```

## 为什么

Context Window 只能保留最近消息，但更早的关键信息不能完全丢。

所以需要 Summary Memory。

它的职责是：

```text

作为长期背景进入后续上下文。
```

注意：

```text
summary 不能替代最近消息。
```

原因：

```text
summary 会丢细节；
最近消息保留原始上下文；
二者要组合使用。
```

```text

```

```python
class Summarizer:

        ...

    def build_prompt(self, messages: list[dict[str, Any]]) -> list[dict[str, str]]:
        ...

    def summarize(self, conversation_id: str, model: str | None = None) -> dict[str, Any]:
        ...
```

---

## Summary Prompt

使用这个 prompt：

```text
请把以下对话压缩成一段会话摘要，用于后续 AI 继续理解上下文。

要求：
1. 保留用户的核心目标。
2. 保留用户正在学习或开发的主题。
3. 保留已经确认的技术方案。
4. 保留关键文件名、函数名、接口路径、错误信息。
5. 删除寒暄、重复确认、无关内容。
6. 不要加入原文中没有的信息。
7. 使用简体中文。
8. 输出一段自然语言摘要，不要输出 JSON。

对话内容：
{messages}
```

### 每条要求的意义

| 要求                  | 意义                        |
| ------------------- | ------------------------- |
| 保留用户核心目标            | 让后续对话不跑偏                  |
| 保留学习/开发主题           | 保持上下文主线                   |
| 保留技术方案              | 防止重复决策                    |
| 保留文件名/函数名/接口路径/错误信息 | 这些是工程上下文中的关键锚点            |
| 删除寒暄                | 减少噪音                      |
| 不加入原文没有的信息          | 防止 summary 自己编造           |
| 使用简体中文              | 和你的学习风格一致                 |
| 不输出 JSON            | summary 是给模型读的自然语言，不是接口结构 |

## 参考实现

```python
# src/app/services/summarizer.py

from datetime import datetime
from typing import Any

from src.app.conversation_store import (
    count_messages,
    get_conversation,
    list_messages,
    update_conversation,
)
from src.app.exceptions import ConversationError
from src.app.services.llm.ollama_provider import OllamaProvider

SUMMARY_TRIGGER_MESSAGE_COUNT = 20
SUMMARY_KEEP_RECENT_MESSAGES = 10


class Summarizer:
    def _ensure_conversation_exists(self, conversation_id: str) -> dict[str, Any]:
        conversation = get_conversation(conversation_id)

        if conversation is None:
            raise ConversationError(
                message="会话不存在",
                detail=f"conversation_id={conversation_id}",
                status_code=404,
            )

        return conversation

    def should_update(self, conversation_id: str) -> bool:
        conversation = self._ensure_conversation_exists(conversation_id)
        total_message_count = count_messages(conversation_id)

        if total_message_count < SUMMARY_TRIGGER_MESSAGE_COUNT:
            return False

        summarized_message_count = int(
            conversation.get("summarized_message_count") or 0
        )

        if not conversation.get("summary"):
            return True

        return (
            total_message_count - summarized_message_count
            >= SUMMARY_KEEP_RECENT_MESSAGES
        )

    def _format_messages_for_summary(
        self,
        messages: list[dict[str, Any]],
    ) -> str:
        lines: list[str] = []

        for message in messages:
            role = message.get("role", "unknown")
            content = message.get("content", "")
            lines.append(f"{role}: {content}")

        return "\n".join(lines)

    def build_prompt(
        self,
        messages: list[dict[str, Any]],
        previous_summary: str | None = None,
    ) -> list[dict[str, str]]:
        messages_text = self._format_messages_for_summary(messages)

        previous_summary_text = ""
        if previous_summary:
            previous_summary_text = f"已有会话摘要：\n{previous_summary}\n\n"

        user_prompt = f"""请把以下对话压缩成一段会话摘要，用于后续 AI 继续理解上下文。

要求：
1. 保留用户的核心目标。
2. 保留用户正在学习或开发的主题。
3. 保留已经确认的技术方案。
4. 保留关键文件名、函数名、接口路径、错误信息。
5. 删除寒暄、重复确认、无关内容。
6. 不要加入原文中没有的信息。
7. 使用简体中文。
8. 输出一段自然语言摘要，不要输出 JSON。

{previous_summary_text}对话内容：
{messages_text}
"""

        return [
            {
                "role": "system",
                "content": "你是一个严谨的会话摘要生成器，只能根据原始对话生成摘要。",
            },
            {
                "role": "user",
                "content": user_prompt,
            },
        ]

    def summarize(
        self,
        conversation_id: str,
        model: str | None = None,
    ) -> dict[str, Any]:
        conversation = self._ensure_conversation_exists(conversation_id)
        messages = list_messages(conversation_id)

        if not messages:
            raise ConversationError(
                message="当前会话没有可摘要的消息",
                detail=f"conversation_id={conversation_id}",
                status_code=400,
            )

        if len(messages) > SUMMARY_KEEP_RECENT_MESSAGES:
            messages_to_summarize = messages[:-SUMMARY_KEEP_RECENT_MESSAGES]
        else:
            messages_to_summarize = messages

        prompt_messages = self.build_prompt(
            messages=messages_to_summarize,
            previous_summary=conversation.get("summary"),
        )

        provider = OllamaProvider()
        selected_model = model or conversation.get("model")

        response = provider.chat(
            messages=prompt_messages,
            model=selected_model,
        )

        summary = response.content.strip()
        updated_at = datetime.now().isoformat()
        total_message_count = count_messages(conversation_id)

        updated_conversation = update_conversation(
            conversation_id,
            {
                "summary": summary,
                "summarized_message_count": total_message_count,
                "summary_updated_at": updated_at,
            },
        )

        return {
            "conversation_id": conversation_id,
            "summary": updated_conversation["summary"],
            "summarized_message_count": updated_conversation.get(
                "summarized_message_count",
                total_message_count,
            ),
            "updated_at": updated_conversation.get(
                "summary_updated_at",
                updated_at,
            ),
        }
```

---

## 逐行解释

### `SUMMARY_TRIGGER_MESSAGE_COUNT = 20`

消息数达到 20 条才考虑自动 summary。

原因：

```text
如果每轮都 summary，速度慢、成本高、还容易不断改写摘要。
```

### `SUMMARY_KEEP_RECENT_MESSAGES = 10`

最近 10 条不摘要，保留原文。

原因：

```text
最近对话细节最重要，不能只靠摘要。
```

### `should_update`

逻辑：

```text
1. 消息不到 20 条，不自动 summary
2. 没有 summary，达到 20 条就 summary
3. 已有 summary，则新增超过 10 条后再 summary
```

### `build_prompt`

它只构造模型输入。

```python

```

新增内部函数：

```python
def _try_update_summary(
    conversation_id: str,
    model: str | None = None,
) -> None:
    summarizer = Summarizer()

    try:
        if not summarizer.should_update(conversation_id):
            return

        summarizer.summarize(
            conversation_id=conversation_id,
            model=model,
        )

        logger.info(
            "会话摘要已更新: conversation_id=%s",
            conversation_id,
        )

    except Exception as exc:
        logger.exception(
            "自动更新会话摘要失败: conversation_id=%s error=%s",
            conversation_id,
            exc,
        )
```

普通接口中，在 `assistant_message = create_message(...)` 后面加：

```python

    conversation_id=conversation_id,
    model=selected_model,
)
```

流式接口中，在 `assistant_message = create_message(...)` 后面加同样代码：

```python
_try_update_summary(
    conversation_id=conversation_id,
    model=selected_model,
)
```

## 逐行解释

### 为什么 try/except

summary 是增强能力，不应该影响主聊天链路。

如果 summary 失败，正确行为是：

```text
聊天正常返回
日志记录 summary 失败
下次还可以重试
```

### 为什么在 assistant message 保存后触发

因为一次完整轮次是：

```text
user message + assistant message
```

保存 assistant 后，summary 才能包含完整问答。

### 为什么流式也要触发

因为流式聊天也是正式聊天。如果流式不触发 summary，那么用户主要使用流式时，summary 永远不更新。
